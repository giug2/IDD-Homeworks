Abstract

Federated Learning has emerged to cope with raising concerns about privacy breaches in using Machine or Deep Learning models. This new paradigm allows the leverage of deep learning models in a distributed manner, enhancing privacy preservation. However, the server‚Äôs blindness to local datasets introduces its vulnerability to model poisoning attacks and data heterogeneity, tampering with the global model performance. Numerous works have proposed robust aggregation algorithms and defensive mechanisms, but the approaches are orthogonal to individual attacks or issues. FedCC, the proposed method, provides robust aggregation by comparing the Centered Kernel Alignment of Penultimate Layers Representations. The experiment results on FedCC demonstrate that it mitigates untargeted and targeted model poisoning or backdoor attacks while also being effective in non-Independently and Identically Distributed data environments. By applying FedCC against untargeted attacks, global model accuracy is recovered the most. Against targeted backdoor attacks, FedCC nullified attack confidence while preserving the test accuracy. Most of the experiment results outstand the baseline methods.

1 Introduction

The widespread use of Machine or Deep Learning (ML/DL) algorithm grows concerns about privacy breaches. Specifically, the development of DL led humans to enjoy technology (i.e., personalized recommendation, efficient and less error-prone computation, and more accurate forecast); however, it requires data to be transmitted and collected in a single location for training which introduces a single point to failure or the risk of data leakage. Along with such concern, stringent privacy rules, such as GDPR, mandate a higher degree of privacy preservation. Accordingly, the need for privacy-protecting mechanisms has risen, and Federated Learning (FL) has emerged.

Federated Learning is a distributed machine/deep learning framework. Each participating end device (client) trains a local model and sends the trained model parameters to a server for aggregation. In other words, data does not leave its origin; it is especially suitable for data-sensitive environments wanting to exploit the advantages of DL while preserving data privacy. A vanilla FedAvg [15] works by repeating the following steps. A server initializes a global model and advertises it to all clients. The server then randomly selects nùëõn clients. The selected clients train the model with their privately-owned data and then upload the model updates to the server. The server aggregates the model updates by averaging them. These steps are repeated until a stopping criterion is met. The fact that the clients only keep their training data locally, instead of transmitting them to a single location, functions as a beneficial factor for data privacy by reducing the attack surface (i.e., a chance of data leakage on the server or in-between communication).

On the other hand, by its distributed nature, FL is vulnerable to model poisoning attacks. The server‚Äôs inaccessibility to local datasets makes it impossible to examine data quality, validate model parameters, and guarantee that the data are Independently and Identically Distributed (IID). Compromised clients or attackers exploit the vulnerability to craft the local model parameters before they send them to the server for aggregation. They aim to indiscriminately hamper the global model performance (untargeted) or make wrong predictions on an attacker-chosen class of inputs (targeted). Targeted model poisoning attacks that maintain their performance on unchosen classes are called backdoor attacks. Backdoor attacks are known to be stealthier and more difficult to detect because the attacker only misclassifies a specific class of inputs as a chosen label while leaving the performance on unchosen ones as usual.

In line with this, a number of robust aggregation methods and defensive mechanisms against model or data poisoning attacks have been proposed. Some preliminary works allow a server to have a portion of raw data to teach it what benign features look like; some allow data sharing between clients to give the server knowledge about how data are distributed and what is normal. Nevertheless, if FL is used with shared data, the peculiar advantages of FL would be significantly diminished and endangering raw data exposure.

Furthermore, previous works mitigating targeted or untargeted model poisoning attacks are primarily in an IID data environment. Nonetheless, data are more likely to be non-IID in practice; previous methods may not work as expected in practical settings. Non-IID data, also known as heterogeneous data, entails a different number of data samples for each class, distinct classes of data for each client, different features but with the same label, or the same feature but with different labels [10]. Due to its unidentical distribution, the participation of clients having heavily non-IID datasets may gradually deteriorate the global model performance as the learning process continues. Nonetheless, even for a server with robust aggregation algorithms, it is difficult to detect and remove malicious clients reliably because the degree of non-IID is directly proportional to the effect of attacks. Thus, the server may mistakenly exclude benign clients from aggregation.

Thus, we develop FedCC that provides robust aggregation under untargeted model poisoning and targeted backdoor attacks while considering the non-IID data environment. FedCC disallows any raw data or additional information sharing between a server and other clients so as to maximize the advantages given by FL. FedCC compares each model‚Äôs Centered Kernel Alignment (CKA) of Penultimate Layer Representations (PLRs) to identify malicious clients over benign ones. FedCC enhances global accuracy against untargeted model poisoning attacks and decreases attack confidence against targeted backdoor attacks while preserving benign accuracy compared to the baseline methods.

Our contributions are summarized as follows:

  • ‚Ä¢

    We develop FedCC that mitigates untargeted model poisoning and targeted backdoor attacks comparing CKA of PLRs of local and global models while taking non-IID into account.

  • ‚Ä¢

    FedCC does not require knowledge about clients or any centralized data for specialized aggregation.

Section 3 defines the threat model, including the attacker‚Äôs goal, capabilities, and strategies. Section 4 introduces FedCC, our robust aggregation method against model poisoning attacks. Section 5 identifies experimental settings, including datasets, model architecture and evaluation metrics. Section 6 illustrates and explains experiment results. Finally, Section 7 concludes the paper.

2 Backgrounds and Related Works

2.1 Poisoning Attacks in FL

FL is vulnerable to poisoning attacks due to its distributed nature and the server‚Äôs inability to directly examine the dataset quality and model parameter integrity. Poisoning attacks are categorized into data and model poisoning attacks based on what the adversaries manipulate. However, model poisoning attacks are more effective and destructive as the malicious client‚Äôs parameters are stealthily tuned to maximize the damage to the overall performance [4, 2], we focus on model poisoning attacks in this paper.

Model poisoning attacks allow client-side adversaries to alter the local model parameters directly before sending them to the server. To avoid divergence from the global model, the attackers optimize the local model for both training loss and an adversarial objective, improving its stealthiness [4, 6]. The adversarial clients can additionally manipulate the model hyperparameter, such as the learning rate, number of local epochs, batch size, and optimization objective, to manipulate the training rules before and during the local training.

Based on the attacker‚Äôs goals, model poisoning attacks are subdivided into untargeted and targeted model poisoning attacks. Untargeted model poisoning attacks diminish global test accuracy, while targeted model poisoning attacks aim to mislead the model to classify a specific class of inputs. In this paper, we added a backdoor ability to the targeted model poisoning attacks, meaning that the inputs of chosen class are misclassified to the chosen label, at the same time, the performance of the rest is not impacted, making the attack stealthier and thus more challenging to detect.

2.2 Robust Aggregation algorithm in FL

There have been numerous attempts to mitigate model poisoning and backdoor attacks through Byzantine-robust aggregation. Most preliminary works assume the number of attackers (mùëöm) is less than n/2ùëõ2n/2, where nùëõn is the number of participating clients in a current round.

For example, Krum [3] choose one of nùëõn received updates whose Euclidean distance to all the remaining n‚àím‚àí2ùëõùëö2n-m-2 updates is the smallest as a new global model. Since this approach selects only one set of updates as a new global model, outliers are ignored and might differ significantly from the average vectors [1]. Furthermore, it only assumes the data follows IID distribution. Median[20] selects the coordinate-wise median of nùëõn received updates as the final result. As the same as Krum, outliers are ignored with this method, but it takes all updates into account. Trimmed Mean[20] excludes the largest mùëöm and the smallest mùëöm values in each coordinate, then calculates the average value of the remaining n‚àí2‚Äãmùëõ2ùëön-2m values. Bulyan[9] is a combination of Multi-Krum and Trimmed mean. It recursively applies Krum to select n‚àí2‚Äãmùëõ2ùëön-2m updates from the total nùëõn updates. It then applies Trimmed Mean to select the n‚àí4‚Äãmùëõ4ùëön-4m updates to obtain the final result. Although their approaches effectively countermeasure various attacks, all previously mentioned approaches except Median require specifying the number of attackers mùëöm in a priori, which is not a feasible assumption in a practical FL setting.

FoolsGold [7], proposed by Fung et al., identifies poisoning sybils based on the model similarity of client updates. Inspired by that the sybils‚Äô crafted model parameters direct toward a more similar direction to each other than between the benign ones, FoolsGold measured cosine similarity between each local model‚Äôs gradients and marked the most similar ones as sybils. However, their method is specifically designed for handling targeted poisoning attacks from a group of clone-based sybils, not an individual malicious client. Thus, it is limited in that it cannot filter a single malicious client because there is no other malicious one to compare the angle to. Also, their experimental result shows that it is only effective in a highly non-IID dataset.

FLTrust [5], proposed by Cao et al., weighs the local clients‚Äô parameters based on a trust score. The trust score is a ReLU clipped cosine similarity retrieved by applying ReLU clipping on the cosine similarities between each local model and the global model‚Äôs parameters. The ReLU clipping step converts the trust score to zero for clients with negative values and ultimately excludes them. Nevertheless, their method is limited in two ways: weight scaling could prevent a model from convergence, and the clients share a small portion of the innocent dataset with a server, risking raw data exposure.

Previous works only take into account summary statistics (mean or median), direction (cosine similarity), or distance (Euclidean) of all weight vectors. Since highly non-IID data distributions look different from each other, they often fail to distinguish between benign clients having non-IID data and malicious clients; benign clients with non-IID data are often classified as malicious. In contrast, we used a kernel-based metric (i.e., CKA) to measure the similarity between global and local models, reliably identifying correlations between differently trained models.

2.3 Data Heterogeneity in FL

Unlike centralized learning, FL could suffer from performance deterioration or unexpected performance due to data heterogeneity or non-IID. IID refers to Independently or Identically Distributed, and non-IID is that does not belong to IID. Although IID is an ideal condition, non-IID is more realistic, considering end devices‚Äô data diversity. Only a few papers [6, 7, 16, 18] experimented with their methods in non-IID settings. For example, [6] set pùëùp as a degree of non-IID, assigning a training instance with label lùëôl to the lùëôlth group with probability pùëùp, where a higher pùëùp indicates a higher degree of non-IID. Fung et al. [7] and Nguyen et al. [16] simulated a non-IID setting by assigning that each client only holds data from one of the classes. Nguyen et al. additionally controlled the degree of non-IID by a concentration parameter, Œ±ùõº\alpha, of Dirichlet distribution, where Œ±=0ùõº0\alpha=0 indicates entirely non-IID, and Œ±=100ùõº100\alpha=100 mimics IID. It is important to measure the effectiveness of the defense method in a heterogeneous data environment because the degree of non-IID is directly proportional to the effect of attacks, making it more difficult for an aggregation algorithm to reliably detect and remove malicious clients [17].

2.4 Penultimate Layer Representation in FL

The penultimate layer is the second last layer (a layer before the softmax layer) of a neural network model, as illustrated in Figure¬†1. Wang et al. [18] discovered that PLRs demonstrate the extraordinary capability to distinguish malicious models from benign ones; benign PLRs follow the same distribution, while malicious PLRs follow a different one across multiple datasets and neural network architectures. Specifically, they demonstrated that PLR distances among benign models are smaller than those of benign and malicious models. Their proposed method, FLARE, assigns a trust score to each client based on pairwise PLR discrepancies defined by Maximum Mean Discrepancy (MMD) among all model updates, allocating lower values to those farther from the benign distribution. The model updates are then scaled and averaged, weighted by each client‚Äôs trust score. However, FLARE is limited in that the server has a subset of raw data, continuously redirecting the global model but might jeopardize data privacy in case the server is compromised.

Centered Kernel Alignment (CKA) [11] is the most accurate similarity metric to measure how similar or different two neural networks are from each other. It compares representations between layers and between differently initialized or trained models. In other words, it would be an effective metric to compare the models trained on highly non-IID data. 3 Threat Model

3.1 Attackers‚Äô Goals

We assume two types of attacks: untargeted model poisoning attacks and targeted backdoor attacks. Both attacks involve client-side attackers that manipulate the model parameters before sending them to the server for aggregation. Untargeted attacks in this paper follow the attacks in [6] (Fang attacks) that aim to degrade the global model accuracy on all classes indiscriminately. Fang attacks are designed to evade robust aggregation rules: Krum and Coordinate-wise Median/Trimmed Mean. A targeted backdoor attack follows [2] that misleads a global model to misclassify the data sample of an attacker-chosen class to another attacker-chosen label while maintaining the accuracy of unchosen classes.

3.2 Attackers‚Äô Capability

  • ‚Ä¢

    An attacker on the client side can control multiple compromised clients.

  • ‚Ä¢

    An attacker has full control over k<n/2ùëòùëõ2k<n/2 clients.

  • ‚Ä¢

    An attacker has knowledge about compromised clients‚Äô data samples, local model parameters, and global model parameters of the previous round, and hyperparameters, such as learning rate, optimization method, loss function, batch size, and the number of local epochs.

  • ‚Ä¢

    An attacker has no knowledge or control over the global server and honest clients.

3.3 Attack Strategy

The aim of the attack is to interrupt the model training so as to drop global test accuracy. The attacks include flipping the sign of the malicious parameters so the parameters direct towards the inverse direction along which the global model parameter should change without attacks. We will refer the attack against Krum to Untargeted-Krum and the latter to Untargeted-Med from here.

4 FedCC: Robust Aggregation against Poisoning Attacks

4.1 Overview of FedCC

The overview of FedCC is illustrated in Figure¬†2. The Federated Learning process starts with (1) a server‚Äôs initialization of a global model. The server then sends a copy of the global model to all nùëõn participating clients, selected out of total kùëòk clients as a probability of fraction Cùê∂C. (2) The clients train each local model based on an individually owned dataset, and then each sends the trained weights to the server for aggregation. Our proposed method comes here. (3) The server extracts the PLRs of the global model and each local model and (4) compares RBF CKA values between them. (5) It then clusters the nùëõn CKA values into two, possibly benign and malicious. (6) Since the server cannot assure whether the clients having higher similarities are malicious, it takes a coordinate-wise average of the clients belonging to the cluster having a greater number of members, following the assumption that the number of malicious clients cannot exceed n/2‚àí1ùëõ21n/2-1. (7) The server then updates the global model of the current epoch and starts the next epoch by sending the updated global model to newly selected nùëõn clients.

4.2 Detailed Design

FedCC aggregates non-scaled selected weights where the selection is based on the measurement of the CKA similarities between global and local models‚Äô PLRs. As described in Algorithm 1 lines 2 to 5, a server first extracts PLRs of the global and local model and measures the kernel CKA between the global PLR and each local PLR (lines 7-9). We chose to compare only PLRs (i.e., not all layers) because it is the most differentiable layer among differently trained models. Also, we chose to use RBF CKA because the similarity differences between non-IID clients are smaller than the non-IID and malicious clients. The similarity values are further clustered using a simple K-means clustering algorithm, where the number of clusters is two: benign and malicious (line 11). Any clustering method can be used as long as it can produce only two clusters.

Regarding the assumption that the number of attackers is less than n/2ùëõ2n/2, we set the clients in the cluster having a less number as suspects (lines 13-18). We assign the index of suspicious clients in suspects (line 19) and then only average the clients‚Äô weights not belong to the suspects (lines 21-31) ‚Äî excluding the suspicious clients‚Äô weights from aggregation. Because weight scaling could prevent the model from convergence even when it reaches optima, we do not scale any weights whether they are classified as benign. We termed our proposed method as FedCC.

Different from Krum and Bulyan, which requires the number of attackers as inputs, and FLARE, which allows the server to have raw data, our proposed method requires neither any knowledge about the clients nor any instance of data, thus taking full advantage of FL in the perspective of privacy preservation. Furthermore, FedCC is effective in a practical situation; FedCC distinguishes malicious from benign clients even when the benign ones have non-IID data.

Penultimate Layer. As mentioned above, the server cannot access local datasets; it instead receives the trained model parameters (weight in this case) of selected clients. Thus, the only way to determine whether a client has malicious intent is to analyze the weights it received. The output of the last layer before the softmax layer takes the end result of the convolution and pooling process and reaches a classification decision. In other words, computed weights of the second last layer contain the essential and indicative information that affects the classification result the most.

To prove that PLR is the most distinguishable layer out of all layers, we measure the distance between two clusters of clients, namely benign and malicious, using dendrograms with a single linkage metric and correlation distance method. The cluster distances of all layers are illustrated in Figure¬†3. Each line refers to the cluster distances under attack in IID or non-IID settings; for example, the line ‚Äômkrum_iid‚Äô illustrates the cluster distance under the untargeted-mKrum attack in an IID setting. Observably, the cluster distance is the largest in PLR among all layers; it means that the degree of differences is the maximum with PLRs. We thus decided to use the weights of the penultimate layer of local models, which contains the most characteristic, refined, and rich-enough information among any other layers, to compare its similarity to the global model.

Kernel CKA. Centered Kernel Alignment [11], or CKA, is a similarity metric to measure how similar or different two neural networks are from each other. It compares representations between layers or differently trained models. It creates similarity kernel matrices in the individual space and compares the similarity matrices to characterize the representation space. It has multiple outstanding abilities; it can learn similarities between layers of different widths, random initialization, invertible linear transformations, orthogonal transformations, and isotropic scaling. In other words, given two trained networks with the same architecture but different initialization, layers at the same depth should be most similar to each other.

Table¬†I summarized the test performance with the CIFAR10 dataset under two untargeted attacks and one targeted attack, separably in IID and Non-IID settings. We compared the test accuracy when applying Kernel CKA, linear CKA, MMD, Cosine similarity, and Euclidean similarity. The values under the column ‚Äôtargeted‚Äô are the combination of the test accuracy and backdoor confidence, where the high test accuracy and low backdoor confidence mean the best performance. We can observe that Kernel CKA resulted in the highest accuracy on multiple experiments; thus, we decided to use kernel CKA to compare the similarities of PLRs.

We cannot explicitly assume the data are whether IID across multiple clients. If the data clients have is non-IID, it could be possible to have large angular or magnitude differences, even if they are benign. We thus want a similarity metric that does not depend on distance, and direction (angle), so we use kernel CKA to measure the similarity between the global model‚Äôs PLR and the local models‚Äô PLRs.

5 Experiments

5.1 Dataset

We use Fashion-MNIST (fMNIST) [19], CIFAR10, and CIFAR100 [12] throughout the experiments. fMNIST and CIFAR10 datasets are associated with labels from 10 classes. fMNIST consists of 60,000 training examples and 10,000 testing examples of 28√ó\times28 grayscale images, and CIFAR10 consists of 50,000 training examples and 10,000 testing examples of 32√ó\times32 color images. CIFAR100 consists of 100 classes of labels with the same training and testing samples as CIFAR10. Each dataset is divided into testing and training datasets. The training dataset is equally divided and distributed across the clients for an IID scenario. For a non-IID scenario, the training dataset is distributed based on Dirichlet distribution with a concentration parameter Œ±ùõº\alpha as 0.2. The neural network model architectures for each dataset are specified in Table¬†II, Table¬†III, and Table¬†IV, respectively. FL is constrained by the bandwidth of the network and the computational capability of end devices; thus, we used a lightweight CNN architecture, LeNet, for experiments with CIFAR100. We trained the model from scratch instead of using a pre-trained model provided by the PyTorch library.

5.2 Experimental Setup

We implement the attacks, defense baselines, and FedCC on the Pytorch platform. We implemented three types of model poisoning attacks: two untargeted attacks as in [6] and a targeted attack as in [2], and defenses, including Krum[3], Coomed[20], Bulyan[9], FLARE[18], as baseline for comparison. The number of total clients (nùëõn) is 10, and the fraction of participation is 1.0, as otherwise stated. The number of attackers is three in untargeted attacks and one in targeted attacks as in [18]. Under untargeted attacks, the number of local epochs is five for all clients. Under the targeted backdoor attack, the number of local epochs in benign clients is three, and six for the malicious clients. Each client trains its local model (i.e., [18]) with learning rate of 0.001 using an Adam optimizer. All results are the average values after running each experiment three times.

5.3 Evaluation Metrics

We used confidence and global model accuracy as evaluation metrics. Backdoor attacks aim to misclassify an attacker-chosen label for an attacker-chosen class of data samples, while the unchosen ones are classified normally. confidence measures the likelihood that the output of the model is misclassified; the value is in the interval [0,1]01[0,1]. The accuracy, in the interval [0,100]0100[0,100], is also measured to show that the global model‚Äôs test accuracy is preserved while the backdoor tasks are being eliminated. Since the attacker launching untargeted attacks aims to deteriorate the global model accuracy, the evaluation metric is the global model test accuracy alone.

6 Results and Discussion

6.1 Non-IID Data Environment

We standardized non-IID settings using Dirichlet distribution. We create a disjoint client training data according to Dirichlet distribution with concentration parameter Œ±ùõº\alpha to 0.2, where 0 indicates the most heterogeneity and 100 mimics homogeneity [13]. Œ±ùõº\alpha controls the degree of non-IID; the smaller Œ±ùõº\alpha is, the more likely the clients hold examples from only one randomly chosen class.

6.1.1 Untargeted Model Poisoning Attacks

Table¬†V shows the test accuracy of the global model on three datasets (fMNIST, CIFAR10, and CIFAR100) under Untargeted-Krum and Untargeted-Med in a non-IID environment. Interestingly, the robust aggregation algorithms often result in lower accuracy than when the server only takes an average (i.e., FedAvg). This is because of inaccurate filtering of malicious clients; for example, indeed malicious clients are wrongly marked as benign, so their weights are aggregated. In the upper section (i.e., Untargeted-Krum), we can observe that the accuracy has dropped by a large margin when applying Krum and Bulyan. This is obvious as Untargeted-Krum is explicitly designed to degrade the accuracy when Krum is applied. Coomed cannot effectively mitigate this attack because median values of the weights often fail to represent benign clients, especially in non-IID settings. We also presume the relatively low accuracy of FLARE is because it scales weights even when the model is already showing good performance. Instead, FedCC selectively averages weights, neither boosting nor minimizing. FedCC significantly outperforms the other methods in the experiments with CIFAR100. Some methods like Coomed and FLARE result in better test accuracy meaning somewhat mitigation against the attack; however, as shown, the accuracy is almost twice higher with FedCC. FedCC shows the highest accuracy (66.27% for fMNSIT, 37.40% for CIFAR10, and 14.51% for CIFAR100).

The lower section (i.e., Untargeted-Med) is a different story. The accuracy of FedAvg dropped by a large margin because this attack makes the median values deviate from the direction if there is no attack, making outliers. It also explains the relatively high accuracy when Coomed and Bulyan are applied ‚Äì coordinate-wise median and trimmed mean ignore the outliers. However, FLARE shows a similar trend to the above attack due to the weight scaling. Commonly for both attacks, we can observe that Krum is less robust than Coomed, and Bulyan is less robust than Coomed. It is because Krum picks one local model as the global model, but Coomed aggregates multiple local models to update the global model. Also, Bulyan takes averages of multiple chosen clients, thus having more probability of the malicious model being selected, especially when the benign clients have non-IID data.

FedCC outperforms the other methods showing the highest accuracy (72.76%, 42.37%, and 16.12%, for fMNIST, CIFAR10, and CIFAR100, respectively), even though it does not share any raw data with the server or the other clients. We assume FedCC mitigates both attacks the best because 1) PLRs have the essence of refined information, and 2) the CKA similarity is high for those between the benign clients having non-IID data shard than between the benign and malicious clients.

6.1.2 Targeted Backdoor Attacks

Table¬†VI summarizes the test accuracy, under targeted backdoor attacks in a non-IID setting. We also plot the confidence of the backdoor task in Figure 4 to show how well we minimize the confidence of the backdoor task. We also include the results of baselines.

As shown in Table¬†VI, Krum, choosing one client‚Äôs weights as a global model, is less robust than the other methods aggregating multiple clients‚Äô weights. Coomed and Bulyan, which employ coordinate-wise median and trimmed mean with multi-Krum, respectively, result in relatively higher test accuracy. FLARE lower the main test accuracy due to weight scaling, similar to the trend in Untargeted attacks. Nevertheless, FedCC shows the highest test accuracy, indicating that filtering malicious clients via CKA similarity comparison is more accurate than relying on first-order statistical methods, such as mean or median.

Also, not to lose generality, we take averages of the results from three experiments in the same setting. As shown in Figure 4, FedCC nullifies the backdoor confidence the most. We assume it is the advantage of leveraging CKA, which can calculate the similarity value even when the models are trained on different datasets (different data distribution, in this case) and were initialized differently.

It is true that the other baselines, such as Bulyan, also reduce confidence; however, we argue that FedCC preserves or recovers the test accuracy the most due to the precise discrimination of malicious clients. Also, the steady low confidence indicates that FedCC does not depend on the number of samples or how data are distributed across clients.

6.2 IID Data Environment

In an IID setting, we evenly divide the training dataset among all clients so that the distribution of the data each client has are identical and of the same size.

6.2.1 Untargeted Model Poisoning Attacks

Table¬†VII shows the test accuracy of the global model on three datasets under untargeted attacks in IID settings. The trends seem similar to that of non-IID experiments, but the test accuracy values are a bit higher as the clients have IID data. In the upper section of the table (i.e., Untargeted Krum), Krum and Bulyan result in relatively low accuracy because the attack explicitly targets to interrupt Krum and its variety. When applying Coomed, the accuracy is increased by a large margin because the median values can represent benign clients when every client has an evenly distributed data shard. FLARE could not continuously guarantee well-recovered accuracy because weight scaling constantly changes the weight even if the model is optimized to the minimum. Since FedCC does not scale weights, the model is solely optimized to the minimum by averaging the selected clients. It thus results in the highest accuracy (89.23% for fMNIST, 67.12% for CIFAR10, and 14.51% for CIFAR100).

Also, the accuracy as a result of FedAvg significantly decreased when the Untargeted-Med attack was launched. It drops by a considerable margin because outliers stand out when the clients have IID data shards. Notably, Krum, Coomed, and Bulyan have similar values of accuracy, indicating that the coordinate-wise median values are close to the value least deviating from the weights of the global model. Meanwhile, FedCC shows the highest test accuracy, as 88.44% for fMNIST, 64.74% for CIFAR10, and 17.83% for CIFAR100.

The accuracy increment is the most noteworthy in experiments with CIFAR100: 66% and 58.7% of growth than the second highest one, under untargeted-Krum and untargeted-Med, respectively. Thus, we can conclude that FedCC effectively mitigates untargeted model poisoning attacks when three malicious clients are attempting to reduce test accuracy.

6.2.2 Targeted Backdoor Attacks

Table¬†VIII and Figure¬†5 are test accuracy and backdoor confidence against the targeted backdoor attack in an IID setting, respectively. Nine benign clients and one malicious client participate in the training process, and all their weights are sent to the server for aggregation. Since targeted backdoor attacks stealthily train the global model to misclassify an attacker-chosen class to an attacker-chosen label while the remaining classes are classified normally, test accuracy has not been significantly dropped for all datasets. However, we can observe that the confidence, the likelihood of being misclassified, has been reduced to almost zero with FedCC, invalidating the backdoor task. The confidence of attack of FedCC is much lower than those of FedAvg and Bulyan, which showed better test accuracy for the CIFAR10 dataset. Maintaining the test accuracy is important, but if the defense method cannot considerably reduce the attack confidence, it is difficult to say that the attack is effectively mitigated. Thus, we could conclude that our proposed method does reach the highest test accuracy on the fMNIST and CIFAR100 datasets and comparable accuracy on the CIFAR10 dataset while nullifying the backdoor confidence for all datasets.

6.3 Performance in Varying FL Settings

We measured the effectiveness of FedCC in various experimental settings, including different numbers of malicious clients, different total numbers of clients, different fractions of participation, and different numbers of local epochs, in the realistic non-IID setting. We compared our method to FedAvg, Coomed, which is as much effective as FedCC, and FLARE, which measured the similarities of PLRs.

6.3.1 Impact of the Number of Malicious Clients

We examine the effect of different numbers of malicious clients under untargeted attacks (Untargeted-Krum and Untargeted-Med). We fixed the number of total clients and participation fraction for this set of experiments as 10 and 1, respectively. To follow the assumption that the number of attackers is less than half of the participating clients, we examine the test accuracy where at most four malicious clients are involved. Figure¬†6 illustrates the test accuracy of the given experimental setting. FedCC outperforms the other defensive method except for one case, where two attackers launch an Untargeted-Med attack on the fMNIST dataset. We observe that the test accuracy with defense methods gets lower as the number of attackers is greater. Besides FedAvg, we can observe Coomed has dropped by the largest margin between when there is one malicious client and four malicious clients. This is because the greater the malicious client participation, the greater the probability of the median values being closer to malicious. FedCC, on the other hand, does not depend on angle or distance but identifies hidden correlations between different networks, thus more accurate in filtering malicious clients regardless of the number of malicious clients. We can still observe the accuracy degradation as the number of malicious clients gets greater; however, it is the result of fewer clients being aggregated, not because of malicious clients being selected.

6.3.2 Impact of the Total Number of Clients

We analyze the impact of the total number of clients, 100, 60, and 40, where the participation fraction and the proportion of malicious clients are fixed as 0.1 and 0.3. In other words, we set the numbers of malicious clients to 3, 2, and 1, where the numbers of clients being aggregated are 10, 6, and 4, respectively. The test accuracy is summarized in Table¬†IX. The accuracy goes down as the total number of clients gets smaller. This is because of the data heterogeneity. The weights of only four clients are aggregated when there are 40 clients in total, meaning the learned global model is trained on less diverse data or biased. Nonetheless, FedCC achieves the highest accuracy in almost every case. It indicates that the similarity comparison method (i.e., CKA) correctly filters the malicious clients, even if the clients have non-IID data shards, without sharing any data or knowledge with the server.

6.3.3 Impact of Fraction

In this section, we examine the impact of the participating fraction of clients where the number of total clients is fixed. The participating fractions of clients are 0.1, 0.3, and 0.5, such that the numbers of clients being aggregated are 10, 30, and 50, respectively. The proportion of malicious clients is fixed at 0.3, such that the malicious clients are 3, 9, and 15, respectively. The test accuracy under untargeted attacks is summarized in Table¬†X. We can observe that the test accuracy tends to increase as the participation fraction is greater under untargeted-Krum attacks. It implies that the more client participation in a non-IID setting, the more accurate the global model is produced if the attacks are effectively mitigated. Under untargeted-Med attacks, the test accuracy is the highest with FedCC and second highest with Coomed. These indicate that selectively choosing clients based on CKA similarity is more reliable than taking the median value of each coordinate in a non-IID setting.

Throughout the experiment, the global model trained with FLARE did not converge, and the test accuracy fluctuated. The fluctuation is due to the weight scaling while randomly selecting clients. Unlike previous experiments, the server chooses a fraction of the client every round, meaning not-yet-trained models can be selected. In this environment, weight scaling leveraged by FLARE constantly improperly scale the local weights, causing divergence and fluctuation.

6.3.4 Impact of the Number of Local Epochs

To examine the impact of the number of local epochs, we doubled up the number of local epochs such that the local models are trained for six epochs in benign clients and twelve epochs in malicious clients. The test accuracy is summarized in Table¬†XI, and the confidence of backdoor task is illustrated in Figure¬†7. We observe that doubling the number of local epochs brings the convergence of the confidence of backdoor attacks forward while better at maintaining test accuracy (i.e., compared to Figure¬†4), but it also solidifies the impact of the robust aggregation. It is true that Coomed and FLARE significantly reduce the backdoor confidence as well as FedCC; however, the test accuracy has even increased with FedCC due to its precise discrimination of malicious clients.

7 Conclusion

Due to the prevalent use of machine/deep learning and raising concerns about privacy breaches, Federated Learning (FL) has emerged. FL, however, is known for its vulnerability to model poisoning attacks due to its blindness to local datasets, making it impossible to guarantee IID data distribution. In line with this, a number of robust aggregation algorithms and defensive mechanisms have been proposed, but the approaches are orthogonal to various attacks, as well as lack consideration of non-IIDness. The proposed method does consider both untargeted and targeted (or backdoor) attacks simultaneously while demonstrating its effectiveness in non-IID data environments.
