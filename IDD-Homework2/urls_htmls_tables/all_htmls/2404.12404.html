Abstract

Generating realistic synthetic tabular data presents a critical challenge in machine learning. This study introduces a simple yet effective method employing Large Language Models (LLMs) tailored to generate synthetic data, specifically addressing data imbalance problems. We propose a novel group-wise prompting method in CSV-style formatting that leverages the in-context learning capabilities of LLMs to produce data that closely adheres to the specified requirements and characteristics of the target dataset. Moreover, our proposed random word replacement strategy significantly improves the handling of monotonous categorical values, enhancing the accuracy and representativeness of the synthetic data. The effectiveness of our method is extensively validated across eight real-world public datasets, achieving state-of-the-art performance in downstream classification and regression tasks while maintaining inter-feature correlations and improving token efficiency over existing approaches. This advancement significantly contributes to addressing the key challenges of machine learning applications, particularly in the context of tabular data generation and handling class imbalance. The source code for our work is available at: https://github.com/seharanul17/synthetic-tabular-LLM.

2 Related work

In the field of tabular data synthesis, traditional methods have predominantly focused on sampling techniques, such as SMOTEÂ (Chawla etÂ al., 2002), which have been foundational in addressing dataset imbalances. With the advancement of deep-learning models, Variational AutoencodersÂ (Kingma & Welling, 2013) and Generative Adversarial NetworksÂ (Goodfellow etÂ al., 2014)-based models have demonstrated their efficacy in creating realistic tabular dataÂ (Choi etÂ al., 2017; Che etÂ al., 2017; Jordon etÂ al., 2018; Xu etÂ al., 2019). Recently, large language model (LLM)-based tabular dataset generation methods have been studied. However, this fine-tuning process is resource-intensive, requiring extensive training of large models for each dataset. Furthermore, converting tabular data into natural text format can be inefficient, leading to high token usage and computational cost due to repetitive column names and phrases.

Recently, there has been significant progress in prompt engineering research, aiming to utilize the capabilities of LLMs for specific tasksÂ (Brown etÂ al., 2020; Wei etÂ al., 2022; Zhou etÂ al., 2022; Kojima etÂ al., 2022; Guo etÂ al., 2023). 3.1 Efficient tabular data formatting in prompts

This section delves into the meticulous design of the data format used to present tabular data within the prompt, emphasizing its importance in maximizing the efficiency and effectiveness of LLMs in recognizing and preserving the integrity and correlations of features in the data.

Minimizing preprocessing. Our approach eliminates the requirement for data preprocessing, maintaining the original column names and formats. While preprocessing can be applied to the demonstrations if desired, our framework allows minimal preprocessing, significantly reducing the need for the complex data manipulations often required in traditional approaches.

This approach not only lessens the labor-intensive preprocessing work but also helps in maintaining the integrity of the raw data, similar to the approach used in GReaTÂ (Borisov etÂ al., 2022). This strategy is crucial in preserving the authenticity and integrity of the raw data, enabling the seamless inclusion of both categorical and numerical variables without the need for extensive modifications. As a result, the typically labor-intensive preprocessing efforts are significantly reduced.

Additionally, if accurate and available, our method offers flexibility to integrate supplementary information where relevant. This can be achieved by adding line-by-line variable descriptions at the beginning of the prompt or renaming columns, thereby enhancing the LLMsâ€™ understanding of the data. Such adaptability allows our method to handle a diverse range of data types while maintaining the fundamental structure and relationships present in the original dataset. This approach not only simplifies the data preparation process but also ensures that the LLM can effectively interpret and utilize the data in its most authentic and unaltered form.

Descriptive sentence vs CSV-style. Our method utilizes a CSV-style format to represent tabular data within prompts, optimizing token usage for each data value. This format is particularly advantageous given the limited context window of LLMs, as it allows for a higher volume of in-context learning examples within the same token constraints. As shown in Fig.Â 2, a descriptive sentence format typically consumes about six words to represent a value, including redundant words and spaces (e.g., three words, two spaces, and one comma). In contrast, the CSV format, which places feature names on the first line followed by instance values, significantly reduces the word count to approximately two words per value. This reduction in token count per value enhances the LLMâ€™s capacity for processing more data within its limited context window. Additionally, this approach facilitates the LLMâ€™s ability to recognize feature correlations in the original data, thereby generating data that preserves the relational integrity. The clear, sequential enumeration of variables, free from superfluous elements, enables the LLM to more accurately detect patterns and correlations across variables. This precise representation is vital for capturing and retaining the intrinsic connections between variables, avoiding potential confusion or misinterpretation that could occur with a more descriptive, sentence-based format. We empirically demonstrate the significant role of CSV-style prompting in preserving the Pearson feature correlation of the original data in SectionÂ 4.2.4.

3.2 Random-word replacement strategy.

The prevalence of monotonous, low-diversity data patterns in CSV-style formats can significantly hinder the ability of LLMs to discern patterns and produce lengthy data. For example, an instance with â€˜t,t,t,t,t,â€™ (where â€˜tâ€™ stands for â€˜trueâ€™) may exhibit a lack of diversity in a CSV-style layout, as opposed to the descriptive sentence format. To address this, we introduce the random word replacement strategy that remaps the values of variables with randomly chosen character combinations, thereby introducing necessary variability and distinctiveness between variables. Essentially, this strategy is applied to categorical variables, where each unique value of a categorical variable is replaced with a unique alphanumeric string. This ensures uniform substitution across the dataset, maintaining the integrity of the data structure while introducing variation, as illustrated in Fig.Â 3.

To illustrate, consider variable V1 in a dataset, which has two unique values, â€˜tâ€™ and â€˜fâ€™. In our strategy, these values are replaced by distinct three-character alphanumeric strings, such as â€˜JY0â€™ for â€˜tâ€™ and â€˜GGNâ€™ for â€˜fâ€™, as detailed in Fig.Â 3(a). This mapping is consistent across the dataset, meaning every instance of â€˜tâ€™ in V1 is replaced with â€˜JY0â€™, and every â€˜fâ€™ with â€˜GGNâ€™. Similar mappings are applied to other categorical variables, as demonstrated in TableÂ 3(a). This is applied to all data points in the training dataset, as illustrated in Fig.Â 3(b). The transformed dataset is then used as in-context learning demonstrations.

The benefit of using random alphanumeric strings over simple characters such as â€˜tâ€™ or â€˜fâ€™ becomes evident in datasets with monotonous values. In such cases, different variables often share the same symbols, leading to potential confusion for the LLM. This is particularly problematic when the dataset includes a large number of repetitive variables. The use of distinct alphanumeric strings for each categorical value helps to mitigate this issue. For example, replacing â€˜tâ€™ with â€˜JY0â€™ for V1 and â€˜tâ€™ with â€˜E3Râ€™ for V2 ensures that each variable has a unique representation, preventing the LLM from being hindered by monotonous values and contributing to more accurate and diverse data generation. Even though the new alphanumeric values such as â€˜JY0â€™ or â€˜E3Râ€™ do not hold inherent meaning, they effectively represent categorical distinctions as symbols. As a result, LLMs remain effective due to their remarkable ability to identify and utilize patterns within dataÂ (Mirchandani etÂ al., 2023).

3.3 Group-wise prompting method

To facilitate LLMs for in-context learning in synthetic data generation, we introduce a group-wise prompting designed to enhance efficiency and effectiveness. The central idea is to construct prompts in a structured, coherent, and predictable manner, aiming to guide LLMs in generating data that faithfully represents the original dataset under predefined conditions. Our objective is to go beyond the mere enumeration of examples within the prompt, recognizing the importance of this structured presentation in achieving consistent and condition-aligned outputs from LLMs.

We adopt a set-level template, which initiates with a set header, followed by predefined groups. Each group comprises a predetermined number of data samples that share distinct predefined attributes. For instance, as illustrated in Fig.Â 2, a set includes two groups: A with â€œ>>50Kâ€ income and B with â€œ<=<=50Kâ€ income. This set-level template is repeated with diverse data samples in the prompt, reinforcing its structure for the LLM. At the end of the prompt, we add a header signifying the start of a new set but deliberately omit the instances for predefined groups, prompting the LLM to generate them. This last header serves as a cue for the LLM to generate a fresh set of data instances. By providing samples in a group-wise manner, the LLM learns the data characteristics within each group and generates new samples accordingly, maintaining fidelity to the specified structure and attributes of each group.

Our grouping method demonstrates scalability and adaptability, capable of generating data adhering to any specified attribute by modifying the examples within each group. Moreover, the flexibility of the prompt structure, accommodating single or multiple groups, broadens the scope for generating samples as needed.

One intrinsic advantage of our method is its ability in generating instances for both minority and majority values effectively. This stands in contrast to fine-tuning methods, which may tend to overfit to prevalent values, thereby limiting their capacity to generate minority values. Our in-context learning approach ensures a balanced number of examples are provided to the LLM. This balanced exposure ensures that the LLM encounters both minority and majority values in a proportionate manner, allowing it to adequately understand and generate both types of data instances.

Through systematic data organization and the definition of attributes and quantities for each group, our method enables the generation of a predefined number of samples with desired traits. This approach mitigates the risk of the LLM failing to generate any data, fostering consistency and accuracy in the generated datasets. 3.4 Synthetic Data Generation Process

Our tabular data generation process begins by creating a prompt template. This template defines the structure and format of the data we aim to generate. It includes specifications for conditional data generation and optional strategies such as variable descriptions or the random word replacement strategy. Placeholders are included in this template, later substituted with actual data instances from Tğ‘‡T.

Next, we randomly select data instances from Tğ‘‡T. These instances are then transformed into CSV format and integrated into the prompt template, replacing the placeholders. This step provides specific and contextual examples for the LLM. The completed prompt, now containing real data instances in CSV format, is fed into the LLM. The LLM processes this prompt and generates new data, also in CSV format. The generated data is converted back into a tabular format. Here, instances with categorical variable values not present in the training data are discarded to maintain data authenticity.

Until a desired number of samples are obtained, this process is repeated with new instances sampled from Tğ‘‡T. For every prompt, a new set of examples is randomly chosen from each class using uniform random sampling, which mitigates the risk of introducing bias that might occur if a certain subset of data is repeatedly used. The iterative approach allows the LLM to encounter various combinations of original data under specific conditions, leading to the production of diverse samples adhering to the constraints. Although each individual generation may be based on the limited distribution of the given subset, cumulatively, the iterations enable the LLM to explore the entire range of the training data. This method ensures that the final output generated by the LLM reflects a comprehensive range of numerical and categorical values, without requiring extensive preprocessing.

4.1 Experimental Setup

Datasets. We use eight real-world datasets from diverse domains to evaluate our method: Travel, Sick, HELOC, Adult Income, Diabetes, California Housing, Thyroid, and Salary. For binary classification datasets, the minor class is set to be one. To prevent potential data leakage, all duplicate values are removed from the datasets. Subsequently, we randomly split them into 80% training and 20% test sets. The training set is used for generating synthetic data. Additional details for the datasets are provided in AppendixÂ F.

Metrics. For the classification task, we measure the F1 score, sensitivity, specificity, balanced accuracy (BAL ACC), accuracy (ACC), and area under the receiver operating characteristic curve (AUC). We mainly focus on F1 score, sensitivity, and specificity to evaluate classification performance on imbalanced data. For the regression task, mean squared error (MSE), mean absolute error (MAE), and mean absolute percentage error (MAPE) are measured to evaluate the model performance.

Baseline models. In our study, we compare our method with various generative models for tabular data generation, including TVAEÂ (Xu etÂ al., 2019), CopulaGANÂ (Patki etÂ al., 2016), CTGANÂ (Xu etÂ al., 2019), CTAB-GANÂ (Zhao etÂ al., 2021), CTAB-GAN+Â (Zhao etÂ al., 2024), GReaTÂ (Borisov etÂ al., 2022) and TabDDPMÂ (Kotelnikov etÂ al., 2023). This comprehensive evaluation assesses the effectiveness of our approach across different data synthesis paradigms. Furthermore, for the imbalanced classification task, we investigate the effectiveness of our method compared to SMOTE and SMOTENCÂ (Chawla etÂ al., 2002).

Experimental details. For our method, we utilize the OpenAI GPT-3.5 models, specifically the GPT-3.5-turbo-0613 and GPT-3.5-turbo-0613-16k versions. For all experiments including machine learning models for downstream tasks, we employ XGBoostÂ (Chen & Guestrin, 2016), CatBoostÂ (Prokhorenkova etÂ al., 2018), and LightGBMÂ (Ke etÂ al., 2017), known for robust performance. 4.2.3 Ablation study

Classification. We conduct ablation study to assess the impact of the proposed strategies on classification performance. This compares the performance when generated samples are added to the original data. As shown in TableÂ 7, the results indicate that removing the grouping method leads to a decrease in F1 scores within the Travel and Adult Income datasets, dropping from 70.18 to 50.98 and 70.17 to 68.56, respectively. This underscores the effectiveness of Group-wise prompting in mitigating class imbalance, thereby enhancing the F1 score. Due to the presence of repeating values, we adopts random word replacement on the Sick dataset. For the Sick dataset, applying Group-wise prompting actually reduces performance. This decrease is attributed to the LLM stopping generation prematurely due to repetitive patterns such as â€˜f,f,f,f,â€¦â€™, leading to the generation of only the first group and failing to achieve balanced generation. In this case, applying the random word replacement strategy enables balanced generation, resulting in a higher F1 score than applying CSV-style prompting sorely. Using only descriptive sentences for generation significantly reduces the F1 score across all datasets compared to our method, particularly dropping to 0.00 for the Sick dataset. This drop to an F1 score of 0.00 indicates that the generated data is biased towards the major class, resulting in the classifier being trained not to predict the minor class. This implies that generation based solely on descriptive sentences fails to reflect the original data distribution.

The suboptimal performances of the ablated versions indicate that merely applying in-context learning is insufficient for generating high-quality synthetic tabular data. The comprehensive application of the proposed prompting techniques is crucial for generating high-quality synthetic data. Additionally, the fact that using generated samples from the ablated version results in poor classification performance suggest that the improved performance of our method is a result of the modelâ€™s understanding and adaptation, rather than simply memorization of test data from pre-training.

LLM generation efficiency and stability. We analyze the data generation stability of different prompting methods on the Adult Income and Sick datasets. The results are obtained from 100 iterations of inputting 20 random data samples. Here, the grouping organizes samples into two sets, two groups, and five samples per group, resulting in a total of 20 samples. For the calculation of the success rate, we count an inference as successful if at least one data sample is produced, and unsuccessful otherwise.

As shown in TableÂ 8, in comparison with the sentence data format, the CSV-style input significantly reduces the number of input tokens from 1827 to 913 for Adult Income dataset, approximately half, demonstrating the superior token efficiency of the CSV-style over the descriptive sentence data format. In terms of the generation success rate, the sentence format achieved 97%, while the CSV-style only reached 38%. The higher success rate of the sentence format is attributed to the inclusion of additional conditions for generation, whereas the CSV style, generating immediate values, did not incorporate any conditions, leading to a lack of additional data generation. However, when a group-wise prompting is applied to the CSV-style, the success rate increases from 38% to 94%. Using only CSV-style prompting results in significantly low data generation efficiency. Therefore, it is essential to apply group-wise prompting in conjunction to optimize both efficiency and the preservation of statistical relationships in the generated data.

When comparing sample generation quantities, methods without group-wise prompting, including both sentence and CSV-style formats, yield fewer than one sample on average. In contrast, our method greatly surpasses these methods by generating an average of 10.52 samples, which matches our expected number of generated samples for a single set in the groping method. This demonstrates that group-wise prompting effectively controls the quantity of generated data samples, ensuring the stable production of synthetic tabular data. Moreover, the ability to generate multiple data samples in a single inference significantly can lowers generation time and costs.

For the Sick dataset, when group-wise prompting is applied, fewer than the expected 10 samples are generated on average. This is due to the repetitive and monotonous patterns resulting in the generation of only the first group and failing to achieve a balanced generation. However, when employing the random word replacement strategy, there is a notable increase in the number of generated samples, rising from 6.93 to 17.68. This improvement illustrates that our random word replacement strategy significantly enhances the ability of LLMs to generate samples more robustly and effectively.

5 Conclusion

Our approach introduces a simple yet effective prompting method that leverages the in-context learning capabilities of LLMs to generate realistic synthetic tabular data. Our extensive evaluations demonstrate the robustness and effectiveness of our method, covering a variety of baselines and datasets. Specifically, our method consistently improves machine learning performance when combined with the original data across eight public datasets while preserving data distributions and feature correlations. In conclusion, our method marks a significant advancement in synthetic tabular data generation, offering effective solutions for addressing data scarcity and imbalances.

Appendix H More details on the prompt design

This section provides an in-depth explanation of the construction of the proposed prompt. Prompt examples for generating synthetic data are provided in TablesÂ 13-18.

Altogether, the final prompt template for synthetic data generation consists of four parts. Descriptions (Optional). The prompt may start with descriptions of the variables in the dataset, provided line-by-line, if precise and available. These descriptions define what each column in the dataset represents, offering contextual understanding to the LLM about the data. It will be located at the top of the prompt (but not repeated to maintain token efficiency) for additional context.

Set-level examples. Then, a set-level examples are present. This section begins with a data entry format header, specifying the order and names of each feature. This header acts as a guide for the LLM, indicating the start of an example set, which is crucial for interpreting the structure of the tabular data. Following the header, thereâ€™s a line specifying a group name, signaling the start of a specific group. Instances belonging to this group are then listed, one per line, in a CSV style. This process is repeated for each group within the set.

Repetition of set-Level template. To reinforce the data structure and patterns, this set-level template is repeated several times within the prompt, each time with distinct data samples. This repetition is key in enabling the LLM to recognize, learn, and subsequently replicate the set format in its synthetic data generation process.

Trigger for LLM data generation. After all the example sets are presented, the same header that indicates the start of an example set is included at the end of the prompt. This header serves as a signal for the LLM to commence generating synthetic data, maintaining the structure and patterns established in the prompt.

By following this structured approach, the LLM is guided to generate synthetic data that not only adheres to the format and structure of the original dataset but also incorporates variability and distinctiveness in the data patterns. The inclusion of descriptions, careful structuring of set-level examples, and strategic repetition of templates all contribute to the effectiveness of this synthetic data generation method.
