Abstract.

Cross-domain recommendation (CDR) extends conventional recommender systems by leveraging user-item interactions from dense domains to mitigate data sparsity and the cold start problem. While CDR offers substantial potential for enhancing recommendation performance, most existing CDR methods suffer from sensitivity to the ratio of overlapping users and intrinsic discrepancy between source and target domains. To overcome these limitations, in this work, we explore the application of graph signal processing (GSP) in CDR scenarios. We propose \proposed, a unified CDR framework based on GSP, which employs a cross-domain similarity graph constructed by flexibly combining target-only similarity and source-bridged similarity. By processing personalized graph signals computed for users from either the source or target domain, our framework effectively supports both inter-domain and intra-domain recommendations. Our empirical evaluation demonstrates that \proposedconsistently outperforms various encoder-based CDR approaches in both inter-domain and intra-domain recommendation scenarios, especially when the ratio of overlapping users is low, highlighting its significant practical implication in real-world applications.

1. Introduction

Cross-domain recommendation (CDR) has emerged as a promising approach in recommender systemsÂ (Zang etÂ al., 2022). By leveraging additional information sources, it has addressed the long-standing challenges of data sparsity and the cold-start problem. Data sparsity is a prevalent problem in recommender systems that arises because users interact with only a limited subset of the total available items, making it challenging to accurately learn user preferences. Furthermore, recommender systems have difficulty in providing reliable recommendations to newly joined users, so-called cold-start users, due to the absence of historical interaction dataÂ (Man etÂ al., 2017). The core idea behind CDR is to leverage user data from a dense source domain to enhance recommendation performance in a sparse target domain.

There are two main scenarios in CDR, which are intra-domain and inter-domain recommendation (Figure 1). Intra-domain recommendation focuses on recommending items to users within the same domain, while leveraging shared user preferences or interaction patterns to mitigate the data sparsity problem of the target domain. There have been various attempts to tailor domain adaptation methods for facilitating this scenarioÂ (Hu etÂ al., 2018; Li and Tuzhilin, 2020; Cao etÂ al., 2022a; Chen etÂ al., 2018; Huang etÂ al., 2023). On the other hand, inter-domain recommendation aims to recommend items to users in a different domain, where there exist discrepancies between the domains. In this sense, transfer learning methods have been developed to map and align features between the source and target domainsÂ (Zhu etÂ al., 2021; Man etÂ al., 2017; Zhu etÂ al., 2022; Kang etÂ al., 2019). Note that most previous studies on CDR methods have only focused on either an intra-domain or inter-domain recommendation scenario.

Despite their own advantages, existing CDR approaches face several critical challenges. First, the final performance of existing methods largely depends on the ratio of overlapping users, which indicates how many users are shared between the source and target domains, as they train a model by utilizing overlapping users as a bridge to link the two domains. In general, a low overlap ratio impedes the effective transfer of source domain knowledge to the target domainÂ (Chen etÂ al., 2023). Second, most existing methods overlook that two different domains may have fundamentally distinct characteristics and user behaviors, referred to as domain sensitivity, leading to limited performance improvement for CDR scenarios. To maximize the performance gain, the potential helpfulness of source domain knowledge to the target domain should be further investigated.

To tackle these challenges, in this work, we focus on graph signal processing (GSP) as an innovative approach to CDR. Several recent studiesÂ (Shen etÂ al., 2021; Liu etÂ al., 2023a; Xia etÂ al., 2022; Choi etÂ al., 2023) have demonstrated the effectiveness of GSP for collaborative filtering (CF), by seamlessly aligning the concept of GSP with that of CF. Particularly, Graph Fourier Transform (GFT) enables the comprehensive representation of users/items in the spectral domain, offering a powerful method for uncovering underlying patterns in complex graph structures. The existing GSP-based CF methods, such as GF-CFÂ (Shen etÂ al., 2021) and PGSPÂ (Liu etÂ al., 2023a), build a similarity graph for users and items, and then make it smooth with various graph filtering strategies, including linear filter and ideal low-pass filter. By utilizing users-items and users-users similarity information as personalized graph signals, GSP can provide item recommendations to users based on the predicted (i.e., smoothed) signals. Built upon the unique benefits of GSP-based CF, our proposed approach to CDR does not rely on encoder training, making its performance much robust to the ratio of overlapping users between source and target domains.

This paper proposes \proposed, a novel and unified CDR framework based on Graph Signal Processing. The key difference of \proposedover previous GSP-based CF methods is to fuse preference knowledge of both domains into a cross-domain similarity graph, by flexibly combining a target-only similarity graph and a source-bridged similarity graph. To be precise, target-only similarities refer to information inherent to the target domain, while source-bridged similarities enhance the target domain similarities by using the source domain information through inter-domain similarities between the two domains. In particular, for further personalized signal processing, \proposedadopts various strategies to augment the cross-domain similarity graph and produces personalized input signals according to the augmentation strategy. In the end, filtering usersâ€™ signals on the similarity graph, where the signals are separately defined for source domain users and target domain users, \proposedcan serve as a unified recommendation framework for both CDR scenarios.

Our extensive experiments demonstrate that \proposedmostly shows the best accuracy among a range of baseline methods, including encoder-based, GSP-based CF methods, and the state-of-the-art CDR methods, in both scenarios. Specifically, our \proposedframework working on the cross-domain similarity graph outperforms other GSP-based CF methods working on a single domain or unified domain similarity graph; this indicates that \proposedmore effectively utilizes source domain knowledge to enhance target domain recommendation. Notably, for the Amazon dataset known for its high sparsity and low overlap ratio, \proposedachieves a remarkable performance gain of over 20% compared to the best baseline results. Furthermore, \proposedis validated to be less sensitive to the ratio of overlapping users, resulting in enhanced robustness for CDR. For reproducibility, our codes are publicly available at https://github.com/jeongeun/CGSP.

2. Related Work

2.1. Cross domain recommendation

The existing CDR studies can be categorized according to the target CDR scenarios: intra-domain and inter-domain recommendation.

For intra-domain recommendationÂ (Li and Tuzhilin, 2020; Cao etÂ al., 2022a; Liu etÂ al., 2020; Zhao etÂ al., 2020; Yuan etÂ al., 2019; Yang etÂ al., 2024; Li etÂ al., 2023; Xu etÂ al., 2023; Hu etÂ al., 2018; Zhu etÂ al., 2018; Chen etÂ al., 2018), many studies have mitigated the data sparsity problem in the target domain by leveraging information from the source domain. DDSANÂ (Chen etÂ al., 2018) proposes a new domain-similarity regularization to promote domain invariance by constraining the model parameters from different domains to be close to each other. DCDCSRÂ (Zhu etÂ al., 2018) leverages the source domain information along with the sparsity degrees of individual users and items. CoNetÂ (Hu etÂ al., 2018) enables feature learning across domains by incorporating cross-connections in its neural network architecture, which improves the transfer of information between domains. DARecÂ (Yuan etÂ al., 2019) emphasizes the inherent structure of the rating matrix to transfer rating patterns without requiring auxiliary information. CDAMLÂ (Xu etÂ al., 2023) utilizes clustering alongside meta-learning for domain adaptation, aiming to improve recommendations in sparse target domains. CAT-ARTÂ (Li etÂ al., 2023) proposes a contrastive autoencoder to generate a global user embedding based on information from all domains. UniCDRÂ (Cao etÂ al., 2023) employs domain-specific and domain-shared embeddings along with aggregation schemes, which also allows it to handle both two CDR scenarios.

The inter-domain recommendationÂ (Liu etÂ al., 2023b, 2024; Lu etÂ al., 2024; Zhao etÂ al., 2023; Zhu etÂ al., 2019, 2020; Sahu and Dwivedi, 2020; Man etÂ al., 2017; Cao etÂ al., 2022b; Zhu etÂ al., 2022) aims to provide recommendations to cold-start users who have not interacted with the target domain, by leveraging their information in the source domain. Many studies have focused on transfer learning based on shared user features or mapping functions. EMCDRÂ (Man etÂ al., 2017) uses latent representations and mapping functions for knowledge transfer, and SSCDRÂ (Kang etÂ al., 2019) further proposes a semi-supervised learning method to effectively train the mapping function. CDRIB (Cao etÂ al., 2022b) introduces new regularizers based on the information bottleneck principle to build user-item correlations across domains. PTUPCDRÂ (Zhu etÂ al., 2022) proposes a meta-network to generate personalized mapping functions. UniCDR (Cao etÂ al., 2023) can also be applied to recommend inter-domain recommendations using domain-shared embeddings.

Though effective, the existing CDR methods are often largely dependent on the ratio of overlapping users, as they train a model by utilizing overlapping users as a bridge to link the two domains. Also, most CDR methods lack the capability of explicitly adjusting the impacts of the source domain, which makes them ineffective when two different domains have large intrinsic discrepancies in terms of user behavior.

2.2. GSP-based Collaborative Filtering

Recently, Graph Signal Processing (GSP) has gained significant attention as a promising direction for recommender systemsÂ (Shen etÂ al., 2021; Liu etÂ al., 2022a, 2023a; Xia etÂ al., 2024, 2022; Liu etÂ al., 2022b; Choi etÂ al., 2023; Hong etÂ al., 2024), as its primary concept aligns with the goal of collaborative filtering, which ensures that similar users share similar embeddings; this would be further explained in SectionÂ 3.2.

Shen etÂ al. (2021) demonstrates that graph filtering methods are closely connected to existing CF methods. Motivated by the role of linear filter in LGCNÂ (He etÂ al., 2020), they suggested LGCN-IDE, which stands for LightGCN with Infinitely Dimensional Embedding. They also proposed GF-CF, a unified graph convolution framework that employs a mixed-frequency graph filter combining linear and ideal low-pass filter. Building on this mixed filter approach, FIREÂ (Xia etÂ al., 2022) improves real-time recommendation efficiency and model updating speeds, thereby addressing dynamic interactions and the cold-start problem. PGSPÂ (Liu etÂ al., 2023a) further develops this concept concept by incorporating an augmented similarity graph with personalized graph signals. HiGSPÂ (Xia etÂ al., 2024) advanced the framework with a module consisting of cluster-wise and globally-aware filters.

While GSP has proven effective in single-domain recommendations, its effectiveness in CDR remains unexplored in the literature. We believe this presents a promising research opportunity, as applying GSP to CDR could provide novel solutions to inherent challenges of previous CDR methods.

3.2. GSP-based Collaborative Filtering

Recently, there have been several attempts to employ GSP for collaborative filtering (CF)Â (Shen etÂ al., 2021; Liu etÂ al., 2023a). The main concept of GSP is smoothness, which refers to the property of a signal on a graph where connected nodes (representing users or items in CF) have similar signal values. A smooth signal on the graph suggests that users and items connected by a short path within the graph are likely to interact with each other, indicating shared preferences. For instance, the linear filter leverages the spectral properties of the graph, particularly focusing on the lower spectrum of the Laplacian matrix, which corresponds to the more smoothly varying components across the graphÂ (Yu and Qin, 2020). This concept aligns with the goal of CF, which aims to exploit similarities between users and items for recommendation.

Existing GSP-based CF methods involve three steps:

  1. i)

    Constructing a similarity graph of users and items

  2. ii)

    Defining a graph filter for the similarity graph

  3. iii)

    Generating and filtering personalized graph signals

The first step is to build a similarity graph for users and items based on their interactions. The similarity graph encodes the underlying relationships among users, among items, and between users and items. Next, the second step is to apply a graph filter. Existing methodsÂ (Shen etÂ al., 2021; Liu etÂ al., 2023a; Xia etÂ al., 2022, 2024) apply a mixed frequency filter that combines a linear filter and an ideal low-pass filter to capture both locally smooth and globally rough signals, thereby offering a comprehensive understanding of user interactions and preferences.

The last step generates an input signal representing each user and then processes it through the filtered similarity graph, in order to capture meaningful patterns across items; this results in predicted (i.e., smoothed) scores for all items, which are used for recommendation in the end.

4.1.2. GSP-based CDR framework

The most straightforward solution for utilizing user-item interactions in both domains for GSP is to employ a unified graph that merges two graphs with overlapping users. This solution can be effective when the source and target domains share a sufficient number of users. However, when there is minimal user overlap, the limited anchor information severely restricts effective knowledge transfer across domains; this diminishes the benefits of utilizing the source domain. Therefore, we concentrate on developing a GSP-based CDR framework that can adeptly incorporate cross-domain information, enhancing the effectiveness of GSP even when direct user overlap is low.
