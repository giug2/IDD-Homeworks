Abstract

Existing federated learning methods have effectively addressed decentralized learning in scenarios involving data privacy and non-IID data. However, in real-world situations, each client dynamically learns new classes, requiring the global model to maintain discriminative capabilities for both new and old classes. To effectively mitigate the effects of catastrophic forgetting and data heterogeneity under low communication costs, we designed a simple and effective method named PLoRA. On the one hand, we adopt prototype learning to learn better feature representations and leverage the heuristic information between prototypes and class features to design a prototype re-weight module to solve the classifier bias caused by data heterogeneity without retraining the classification layer. On the other hand, our approach utilizes a pre-trained model as the backbone and utilizes LoRA to fine-tune with a tiny amount of parameters when learning new classes. Moreover, PLoRA does not rely on similarity-based module selection strategies, thereby further reducing communication overhead. Experimental results on standard datasets indicate that our method outperforms the state-of-the-art approaches significantly. More importantly, our method exhibits strong robustness and superiority in various scenarios and degrees of data heterogeneity. Our code will be publicly available.

1 Introduction

Federated learning (FL) [35] is a novel distributed machine learning paradigm that enables multiple data owners to collaboratively train a shared model while ensuring the privacy of their local data. In recent years, with the increasing emphasis on data privacy in society and the refinement of relevant regulations (e.g., GDPR) [50], FL has experienced rapid development and has been widely applied in various real-world scenarios  [19, 37, 42].

Existing FL methods [35, 26, 18] typically rely on the closed-world assumption [12], that is, the number of classes seen by the model remains the same and unchanged during both the training and testing stages. However, the real world is dynamic and constantly changing, the local clients often need to continuously receive new data to update the global model. Therefore, Federated Class Incremental Learning (FCIL) [57, 7] has been proposed to cope with FL tasks in dynamic scenarios. Specifically, each local client can only update the model with new class data at each stage and upload the model parameters to the global server for aggregation, while the global model needs to maintain its discriminative ability for all seen classes. Moreover, the data distribution of local data follows the non-independent and identically distributed (non-IID) assumption [25].

FCIL presents a more realistic setting for real-world applications, but it also introduces greater challenges, as FCIL needs to simultaneously address the issue of catastrophic forgetting [34, 22] and data heterogeneity resulting from non-IID. In existing studies, one approach is storing a subset of datas of old classes and training them together when learning new tasks [8, 7], but the amount of old data stored is strictly limited due to privacy protection requirements. An alternative way is to utilize generative models to generate pseudo-samples of old data [38, 59], thus retaining the ability to classify the old classes. However, training a good generator will cause greater computational overhead, and the generator also suffers from catastrophic forgetting. Another direction is to use pre-trained transformer models for fine-tuning [28, 1], but these methods rely on similarity matching between modules and will occupy additional local storage space. Although they provide different perspectives for solving FCIL, there is still a large unexplored space, especially the performance of the model under different non-IID settings and different degrees of data heterogeneity. So in order to better address this issue, we start by addressing a fundamental question:

What is the panacea for dealing with CIL and FL?

In other words, if we can find common ground in dealing with CIL and FL, it will be of great help in solving the FCIL problem. Surprisingly, we combined experimental results (Section 3.2) with related work and found that: 1) At the architecture level, the performance of both FL and CIL depends on the choice of the model architecture. On the one hand, recent works [39, 48] demonstrate that simply replacing the convolutional neural network with a self-attention-based architecture (e.g., Transformers) [49, 9] can effectively improve the model’s performance under data heterogeneously. On the other hand, L2P [52], DualPrompt [51], CODA-Prompt [45] organically combine the Parameter-Efficient Fine-Tuning (PEFT) [14] with CIL, and achieve competitive results than exemplar-based methods in non-exemplar based setting. 2) At the classifier level, classifier drift is the common enemy of both. In CIL, When learning new classes, the decision boundaries learned from old classes can be severely disrupted, resulting in a significant bias in the classification layer [63, 62]; In FL, CCVR [33] shows that the classifier has the lowest feature similarity across different local models, which means that the classifier of each client models is biased to the classes of local data.

Motivated by the above findings, we propose a Prototype and Low-Rank Adaption (PLoRA) model to tackle the FCIL problem. Specifically, We draw inspiration from prototype learning and incorporate prototype classifiers into our model. Compared with traditional cross-entropy-based classifiers, prototype learning captures both discriminative and generative information, which is beneficial for class-incremental learning [54, 55, 32]. To address the classifier bias caused by data heterogeneity, we design a prototype re-weight module for the global server, which aggregates the local prototypes based on heuristic information derived from corresponding prototypes of class features. Considering memory and communication cost, we utilize LoRA [17] to fine-tune the pre-trained model, where the parameters of the backbone network are frozen, and only the parameters of LoRA are trainable and uploaded to the global server. For the choice of pre-training weights, we posit that supervised pre-training weights may trigger privacy issues in FL as the classes seen during the pre-training process may overlap with the test dataset, so we consider both supervised and self-supervised pre-trained weights in our experiments. Compared to other existing FCIL methods, our model achieves superior results on standard datasets. Furthermore, we explore the model’s performance under different non-IID settings and degrees of data heterogeneity. Experimental results demonstrate the robustness of our approach, maintaining stable performance, while other methods experience significant declines. The main contributions of this paper can be summarized as:

  • •

    We analyze the common ground between CIL and FL, pointing out that the choice of model architecture and classifier drift are key issues for both. Motivated by this, we propose Prototype and Low-Rank Adaption (PLoRA) for FCIL.

  • •

    We adopt a prototype classifier and design a prototype re-weight module to effectively solve the classifier drift, and we first explore using LoRA to fine-tune the pre-trained model in FCIL, achieving better performance at low communication cost.

  • •

    We conduct extensive experiments on standard datasets and achieve state-of-the-art performance. Furthermore, to our best knowledge, we are the first to fully explore the performance of the model under different non-IID settings and degrees of data heterogeneity.

2 Related Work

Federated Class-Incremental Learning: Research on FCIL has garnered considerable attention in recent years. Dong et al. [7] first introduces the concept of FCIL and proposes several loss functions on the local side and the global server side to alleviate local catastrophic forgetting and global catastrophic forgetting. However, their method uses a rehearsal buffer to store and retain old class data, and additionally design a proxy server to select the best model, resulting in large memory overhead and communication costs. LGA [8] extends the work of [7], but it still belongs to rehearsal-based FCIL. In the more challenging rehearsal-free FCIL problem, generate models are widely adopted to produce synthetic data that aim to mitigate the catastrophic forgetting on local side and global side [60, 38], but its performance is highly dependent on the quality of synthetic data and will incur additional computational costs. Similar to our work, FedSpace [44] uses prototype-based learning, while they use prototypes to aid in the learning of the model’s representation, rather than directly for discrimination. In contrast to the methods described above, some recent studies [1, 28] combine pre-trained models with FCIL and achieve higher performance at a smaller communication cost. However, they both adopt a similarity-based selection strategy, which causes additional memory overheads. Besides, they all use supervised pre-training weights, whereas we argue that this may pose privacy concerns. In our work, we first utilize LoRA to fine-tune the pre-trained model, resulting in a unified global model, and we consider both supervised and self-supervised pre-training weights.

Prototype Learning: Prototype learning aims to learn the prototype vectors for each class from the feature space. Yang et al. [55] proposes a convolutional prototype network (CPN) to learn intra-class compact and inter-class separable features, which facilitates open set recognition. In CIL, prototypes are used to pseudo old class data to alleviate catastrophic forgetting at the classifier level [63, 64]. To tackle data heterogeneity in FL, some works [36, 47, 6] construct prototype loss to guide the model in learning better feature representations or to convey the data distribution information from local clients through prototypes. Fedproto [46] is similar to our method when dealing with data heterogeneity, but we directly employ prototype classifiers for discrimination and perform heuristic weighted aggregation of the prototypes on the global server.

PEFT for Pre-Trained Model: With the emergence of large-scale pre-trained models [4, 40, 21], how to effectively fine-tune these models to adapt the downstream tasks has been a focal point of attention. Recently, LoRA [17], Prompt [31], and Adapter [16] have emerged as standout techniques, especially prompt has been widely used in CIL [52, 51, 45] and FL [13, 61]. However, whether it is Prompt or Adapter, existing methods require additional pools to store parameters of Prompt or Adapter modules which will undoubtedly cause huge memory overhead for FCIL as the number of clients increases. In this work, we extend LoRA to the FCIL task for the first time, introducing a novel approach that does not require additional memory allocation.

3.2 The intersection of CIL and FL

Typically, the model can be divided into two parts: feature extractor and classifier. In this section, we conduct experiments to illustrate and analyze how feature extractor and classifier affect FL and CIL performance.

3.2.1 Model architecture: Transformers or CNN?

A major difference between CNN and Transformers is that the former relies more on local high-frequency patterns, while the latter can learn global interaction through self-attention [39, 11]. We argue that under FL tasks, the global information learned by Transformers is more robust under data heterogeneity. Fig. 1 shows that the local patterns learned by CNN are greatly affected by data distribution, causing the model to lose some important patterns after averaging, while the global interactions learned by Transformers are significantly more robust.

For CIL tasks, IL2A [62] illustrates that learning transferable and diverse representations helps address representation bias. To obtain better representations, the pre-trained model provide a natural solution [53]. On the one hand, the parameters of the pre-trained model are trained on large-scale data sets rather than randomly initialized, so they provide solid initial values for the model when adapting to downstream tasks, and Transformers are more suitable for training on large-scale datasets compared to CNN. On the other hand, fine-tuning pre-trained models with a minimal number of parameters through PEFT has been widely used to enhance the performance of CIL [52, 51, 45].

Considering the above two points, the Transformers architecture has natural advantages over CNN in handling FL and CIL tasks. Therefore, we use the pre-trained Transformers architecture model (ViT) as the backbone network to address the FCIL problem.

3.2.2 Classifier bias: the common enemy of both

In CIL, classifier bias means that when learning a new task, the discrimination boundary that has been learned in the past will drift seriously, thereby losing the ability to discriminate old classes. To tackle this issue, Deesil [2] and IL2m [3] directly store part of the old class data for subsequent training; PASS [63] retain pseudo features of old classes to assist in training the classification layer. However, we find that recent works adopt a simple but effective trick of not updating the parameters of the old classes classification layer when learning new tasks, thereby retaining the model’s classification ability for old classes, especially methods based on pre-trained models [32, 52, 51]. We argue that the reason this trick has not been explored before is that features of old classes drift severely when learning new tasks, so even if their classifier parameters are preserved, the feature-classifier mismatch can cause catastrophic forgetting.

As for FL, [33] observe that the classification layer has the lowest feature similarity among all layers, which indicates that the classifier of the local model will severely fit the distribution of local data. Similar to CIL, [33, 43] utilize the statistical information of local features to retrain the classifier on the global server to alleviate classifier bias. However, this approach also incurs additional computational overhead. In this paper, we initially employ prototype learning and introduce a generative loss to better model the distribution of features. Additionally, we design a simple yet effective prototype re-weight module to efficiently address classifier bias.

4 Our method

4.1 Low-Rank Adaption for Pre-trained Model

Though methods based on Prompt [1] and Adapter [28] have demonstrated robust performance in FCIL, each local client of these methods still requires memory for storing prompt or adapter module parameters. As the number of clients increases, the system’s storage capacity and communication costs also escalate. Furthermore, they must calculate the similarity between input samples and modules to choose the appropriate module, especially in  [28], where an additional pre-trained CNN network is required for module selection, which undoubtedly consumes more storage space and the performance heavily relies on the quality of module selection. Hence, we propose using LoRA [17] to fine-tune pre-trained models without the need for additional module selection.

4.2 Prototype learning with Prototype Re-weight

Therefore, the model not only needs to correctly classify known classes but also needs to model the distribution of known classes in the feature space. In open set recognition, CPN [55] designs a discriminative loss and generative loss for prototype learning to constrain the range of known classes in the feature space, thereby reserving space for samples from unknown classes. 4.3 Integrated Objective of PLoRA

When the local model learns new tasks, the features of old classes undergo feature drift. Therefore, to reduce the mismatch between old class prototypes and features caused by feature drift, we employ knowledge distillation (KD) [15, 41] to constrain the changes in the feature extractor. Inspired by [56], We calculate the following mimicking loss on the features of the middle layer (layer 5) of the global model and the local model during training:
