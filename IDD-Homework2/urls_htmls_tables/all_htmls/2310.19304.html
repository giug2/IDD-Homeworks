0.1 Graph Based Analytics

Automatically identifying suspicious financial activity in a large set of transactions is a notoriously difficult problem, as the intent of the transactions is not explicitly provided and must be inferred, often without the benefit of any external information. While malicious behavior is uncommon, it is often similar to benign (and equally anomalous) behavior. Thus, we are stuck with a low signal to noise environment with a high class imbalance when training machine learning recognizers. Furthermore, malicious actors can vary their behavior over time to avoid detection. To assist in overcoming these difficulties, we organize the transaction data into a graph where nodes represent accounts (transaction end points) and edges represent the transactions between accounts. Armed with this representation, we can use knowledge of previously identified suspicious transactions to inform an opinion about the transacting accounts and use that to in turn bias judgement about more ambiguous transactions involving those accounts, thus propagating the “taint” thru the network. We can also apply efficient algorithms to recognize patterns in the graph topology that are commonly associated with suspicious behavior. We leverage the transaction graph both to compute statistical features of graph nodes and edges, including their attributes and neighborhoods, and to drive graph based machine learning models, like Graph Neural Networks (GNN’s). In our prior work, we have found combining these approaches generates meaningful lift. The transaction data (PNS), as received, is first converted into node/edge list format and then loaded into the graph store which serves as a platform for machine learning feature generation. The graph store platform supports the efficient computation of graph based statistical features by enabling rapid access to adjacent nodes and edges as well as exploitation of parallel processing by utilizing a message passing programming model. Furthermore, the historical data and statistics allows the generation of feature values requiring comparison to a historical base, such as sender average amount, or distribution over time of day when money is sent. Features generated here are consumed by the machine learning models (GNN and Random Forest) during both the training and inference phases. Graph Neural Networks (GNNs) are a powerful machine learning technique that can capture information about the context (surrounding nodes and edges) of a node in a graph, much the way a Convolutional NN captures information in an image from surrounding pixels. For the PNS transactions, we can exploit these abilities to recognize patterns of suspicious activity comprised of sets of related transactions and develop a bias as to whether a given transaction is suspicious that can be integrated (as a feature) into the overall privacy preserving decision trees that incorporate non-private (PNS) and private (bank) information. We choose this approach over simply ensembling the GNN output with the trees because a negative value from the GNN is less informative due to the absence of private information from the banks. The detailed design of the GNN is not discussed here due to space limitations, but required characteristics include inductive learning (generate models applicable to new nodes/edges not in training), and scalability thru sampling of nodes, and ability to limit computational explosion when processing high degree nodes (power law distributions are common in graph data, including transactions)[jchen1]. \includegraphics[width=]imgs/GraphPipelineDiagram.png
