Diversifying Human Pose in Synthetic Data
for Aerial-view Human Detection

Yi-Ting Shen   Hyungtae Lee   Heesung Kwon   Shuvra S. Bhattacharyya 11**22**3311
Abstract

We present a framework for diversifying human poses in a synthetic dataset for aerial-view human detection. Our method firstly constructs a set of novel poses using a pose generator and then alters images in the existing synthetic dataset to assume the novel poses while maintaining the original style using an image translator. Since images corresponding to the novel poses are not available in training, the image translator is trained to be applicable only when the input and target poses are similar, thus training does not require the novel poses and their corresponding images. Next, we select a sequence of target novel poses from the novel pose set, using Dijkstra’s algorithm to ensure that poses closer to each other are located adjacently in the sequence. Finally, we repeatedly apply the image translator to each target pose in sequence to produce a group of novel pose images representing a variety of different limited body movements from the source pose. Experiments demonstrate that, regardless of how the synthetic data is used for training or the data size, leveraging the pose-diversified synthetic dataset in training generally presents remarkably better accuracy than using the original synthetic dataset on three aerial-view human detection benchmarks (VisDrone, Okutama-Action, and ICG) in the few-shot regime.

1 Introduction

Despite the recent explosive use of flying devices, aerial-view detection has not made much progress compared to general detection that has been successful in computer vision, such as R-CNNs [44, 26], DETRs [9, 68], YOLOs [43, 58], etc. This may be due to the lack of real-world datasets that sufficiently capture diverse object appearances (especially humans) in aerial views.

One solution to dealing with the data scarcity issue in aerial-view human detection is to use synthetic data generated by controlling the factors closely associated with large variations in human appearances. For example, several attempts have been made to synthesize images of human postures by employing multiple virtual characters with various outfits or external physical characteristics [5, 60], or by capturing humans from various aerial views at densely sampled viewing angles and ranges [25, 48]. Particularly, human poses, one of the main factors linked to distinctive human appearances, are highly diverse and only partially defined, making them exceptionally difficult to control when constructing a synthetic dataset.

In this paper, we aim to diversify an existing synthetic dataset instead of simply synthesizing more images from scratch. Specifically, we want to transform images in an existing synthetic dataset so that the characters assume poses that do not appear in the original set (‘novel poses’), while maintaining the original style. To make our goal feasible, we need to find appropriate solutions to the following two main questions: i) how to find a variety of novel yet realistic poses, and ii) how to generate human images assuming novel poses.

To address the first question, we introduce a pose generator based on the diffusion model, an emerging generative model that exhibits remarkable performance [52]. The pose generator is designed to provide a variety of realistic human poses in the form of 3D keypoint locations. Using the pose generator, we construct a large set of diverse novel poses (‘novel pose set’) while ensuring a certain level of diversity by not allowing highly similar poses in the set.

To address the second question, we use a source-to-target image translator to create human images such that virtual characters assume various poses in the novel pose set. Ideally, training the image translator requires pairs of source and target images along with their corresponding poses. However, target images corresponding to poses in the novel pose set are not available in our scenario. To deal with this issue, our strategy is to first train the image translator on image pairs from an existing synthetic dataset with diverse viewpoints and fairly limited types of similar poses. The image translator is then applied in a cascaded manner, transforming input poses only to similar yet distinctive target poses. Specifically, we apply the image translation to sequences of target poses (‘target pose sequences’) that demonstrate a variety of limited body movements, gradually transforming an input pose into a final target pose.

Our strategy requires any two adjacent poses in a target pose sequence to be as close to each other as possible. To ensure this property, we first randomly select a few final target poses from the novel pose set for each pose in the original synthetic dataset that serves as the starting pose of each target pose sequence. Then, finding a target pose sequence can be viewed as a problem of finding the optimal path from a starting pose to a final target pose in a graph built on the novel pose set. We use Dijkstra’s algorithm [13] to solve this optimal path search problem. Then, for each target pose in a target pose sequence, a corresponding target image is generated by the image translator using the previous target image as the source image for the current image generation.

With the solutions to the two main questions, we present a framework of constructing a pose-diversified synthetic dataset for aerial-view human detection (Fig 1). We show that leveraging the pose-diversified synthetic dataset in training remarkably enhances the detection performance, especially in the low-shot setup, where the need for additional data is relatively huge. Moreover, regardless of how to leverage synthetic data in training (either ‘pre-training and fine-tuning’ or ‘PTL (Progressive Transformation Learning) [47]’), using the pose-diversified synthetic dataset generally yields significantly better detection accuracy than using the original synthetic dataset on three aerial-view human detection benchmarks: VisDrone [67], Okutama-Action [4], and ICG [1]. The positive impact of using the pose-diversified synthetic dataset continues regardless of the dataset size, supported by a series of scaling behavior analyses of the synthetic dataset from various aspects.

2 Related Works

Synthesizing human appearances. Many efforts have been made to manipulate factors that affect human appearances, such as exterior/clothing features [5, 60], viewing angle and direction [25, 48], human-object interaction [51], to generate diverse synthetic data. The diversity of these factors can be represented by a finite number of parameters respectively. Diverse synthetic data is then created by sampling these parameterized factors.

However, human poses are exceptionally diverse that it is difficult to select representative samples to cover the entire pose space. Therefore, there have been attempts to directly capture human poses from various real human movements. For example, some of these attempts [19, 46, 39] used commercial body scanners (e.g., MoCap [57, 42] or EM sensors [24, 23]) to acquire high-quality and textured human poses in controlled environments. There have also been attempts to capture and synthesize real human poses from 2D images using 3D information (e.g., depth [14] or point clouds [65]).

Similar to our objective, several previous works have altered existing human images to assume diverse poses. For instance, HUSC [64] and SURREAL [55] introduced image translators that altered an image to have different poses that did not present in the original dataset. PoseTrans [21] applied human parsing and affine transformations to existing human images to generate new images with diverse poses. Rogez and Schmid [46] stitched multiple images showing only part of human body to follow a given novel pose. In this paper, we introduce an image translator that addresses the same problem where no ground-truth images of novel poses exist.

Pose generation. In previous works, novel poses can be created by completing missing human keypoints using visible joint information or by sampling from a pose prior distribution. For the pose completion task, several attempts have been made to adopt MLE [29], mixture density regression models [54], or transformers [36, 15, 62]. For the task of sampling from a prior distribution, the pose prior distribution has been captured from human pose datasets in a variety of ways, such as learning multi-task representations [11], constructing an over-complete pose dictionary [2], fitting into a mixture of Gaussian [6], learning a VAE [40], and learning a neural distance field [53]. Also, [12] used adversarial learning to guide an existing pose generator to generate plausible poses. Extending from the second line of tasks, we designed a pose generator based on the diffusion model.

Generating images of novel poses. Many efforts have been made to transform human images so that they can present certain target poses. These efforts adopted various forms of generators, including conditional GAN [30, 61], NeRF [17, 20], and neural feature fields [10], and continued to present better generation quality than previous work. To obtain better generation quality, patch-level disentanglement [30], in contrast to larger-sized part-level disentanglement that inevitably introduced artifacts, and image revision [63], which compared adjacent images in a video sequence, have also been introduced. To address the case where pairs of images corresponding to source and target poses were not available for training, DPE [38] used a bidirectional cyclic training strategy that minimized the distance between two images (including the same character) after transforming them to have the same pose and expression. This strategy was effective only under a strict constraint that required optimal face reconstruction, which is not easily met in our scenario. To address this problem, our strategy is to train an image translator to acquire a limited capability to perform transformation and apply it only when the source and the target poses are close to each other. Specifically, we construct the target pose sequence where adjacent target poses are sufficiently similar and generate the target images in order.

Exploiting various poses in recognition tasks. There are two categories of approaches that consider various human poses for recognition tasks: learning pose-aware models and learning pose-invariant models. In this section, we cover learning pose-invariant models, the category to which our method belongs.

In the past, attempts have been made to develop pose-invariant features that can be used to represent humans, such as articulation-insensitive features [32], bag-of-pose-activation (poselet) [8, 7, 35, 27, 28], and features when taking a standard pose [66]. After the recent introduction of large-capacity deep neural networks, there have been efforts to learn representations to capture human poses appearing in large-scale human datasets. For example, 3DInvarReID [33] learned pose-invariant representations to acquire the reconstruction ability between two different poses. In this paper, our objective is to learn a pose-invariant representation using a pose-diversified synthetic dataset.

3 Method

Our approach to constructing a pose-diversified synthetic dataset has three main components (Fig. 1): pose generator, image translator, and target pose selection. The details of these components are elaborated below:

Pose generator. The pose generator is designed to generate novel yet realistic poses that do not exist in existing datasets. The generated pose takes the form of 3D locations of K𝐾K human keypoints, i.e., 𝐩∈ℝK×3𝐩superscriptℝ𝐾3{\bf p}\in\mathbb{R}^{K\times 3} (in our setting, K=17𝐾17K=17). To acquire such a capability, the pose generator adopts the diffusion model, an emerging generative model that exhibits remarkable performance (e.g., image [49], human motion [52], etc.).

Using the pose generator, we construct a novel pose set 𝒫nsubscript𝒫𝑛\mathcal{P}_{n}. To ensure the diversity of the set, we do not include pose pairs with high similarity in the set. In practice, a newly generated pose is added to the set only if there are no similar poses in the set. Here, the two poses with pose distance (eq. 4) less than a certain threshold Tsimsubscript𝑇simT_{\text{sim}} are considered similar (in our setting, Tsim=0.24subscript𝑇sim0.24T_{\text{sim}}=0.24). This process is repeated until Npossubscript𝑁posN_{\text{pos}} poses are included in the set. Fig. 2 presents some examples of poses in 𝒫nsubscript𝒫𝑛\mathcal{P}_{n}.

Image translator. Training the image translator requires source poses, target poses and their corresponding images. In this work, input poses and novel poses serve as source poses and target poses, respectively. Clearly, target images corresponding to novel poses are not available for training in our scenario. Our strategy to address this issue is to train the image translator to acquire a limited capability that can generate target images only when target poses are similar to source poses. Such limited capability can be achieved by using limited pose pairs from the original synthetic data for training without relying on novel poses in 𝒫nsubscript𝒫𝑛\mathcal{P}_{n}. After that, human images assuming novel poses that differ significantly from the input pose can be generated by repeatedly applying the image translator.

The image translator is only applied to human regions segmented from the input synthetic image to prevent non-human regions adversely influencing the generation. The human region segmentation is performed using segmentation masks, which are generally available along with synthetic data. The segmented human region is refilled in the remaining input image using background inpainting ([50] is used in this work). The generated human images are resized at random ratios and placed at random locations in the background images. The random ratio follows the human-size distribution of the original synthetic dataset.111Details of the image generation process are provided in the supplementary material. Fig. 3 presents some example of generated images.

In this work, the NTED model [45] is adopted as the image translator. It is a recently introduced generative model that can synthesize human images to follow target human poses. It takes an reference human image and a target pose as input and synthesizes an image where the reference human is morphed into the target pose. Although a specific image translator is used, the proposed pose-diversifying framework is not restricted to a particular image translator.

Target pose selection. The image translator lacks the ability to generate images with poses that are significantly different from the input pose. To ensure the diversity of generated images while considering the limitation of the image translator, we introduce a progressive pose search that finds sequential target poses evolved from the input pose.

The progressive pose search adopts Dijkstra’s algorithm [13], which searches for the optimal path from the start node to the destination node in the graph. In our scenario, the novel pose set is treated as a graph, where the input pose and the target pose serve as the start node and the destination node, respectively. The input pose is estimated in the form of 2D human keypoint locations by applying ViTPose [59] to the input image. Post-processing: noisy image filtering, occlusion insertion. For the first post-processing, we consider filtering out noisy images that provide a negative impact on the performance. Specifically, we conduct a sanity check based on pose estimation to filter out noisy images from all generated images. The generated images are considered to have a high probability of being low-quality if their estimated poses are significantly different from the target poses used during the generation process. Accordingly, the generated images whose pose distances (eq. 4) between the input target poses and the estimated poses are greater than a threshold Tfiltsubscript𝑇filtT_{\text{filt}} are not included in the novel image set (in our setting, Tfilt=0.1subscript𝑇filt0.1T_{\text{filt}}=0.1). Samples of noisy generated images are shown in Fig. 4.

For the second post-processing, we consider occlusions, which account for much variation in human appearances in real-world aerial-view images. Among various situations that cause occlusions, we consider the simple and frequently occurring case where a human is occluded by another human. That is, we place two generated humans that share the same camera position in one image. The locations of both humans in the image are randomly determined, but we ensure that there is an overlap between the two humans. Besides, the occluding human is always positioned lower than the occluded human in the image to follow the natural occlusions in the aerial view. Samples of generated images after post-processing are shown in Fig. 5.

4 Experiments

Tasks and datasets. In aerial-view human detection, images are captured by flying devices at various altitudes, ranges, and viewing angles, resulting in very diverse human appearances. Therefore, there is a great need for a large-scale dataset that can sufficiently cover the diverse human appearances. Following [47], we perform evaluations in two data-scarce regimes, the low-shot and cross-domain setups, where the demand for synthetic data is particularly greater.

We use VisDrone [67], Okutama-Action [4], and ICG [1] as the real aerial-view human detection datasets, and use Archangel-Synthetic [48] as the original synthetic dataset. For the cross-domain setups where training and test sets are drawn from different datasets, the VisDrone training set is used for training and the other two datasets are used for testing. The VisDrone test set is also used for testing to showcase the same-domain setup. For the low-shot setups, we use either only 20 or 50 real images randomly selected from the training set for training, denoted as Vis-{20, 50}. To mitigate the effects of randomness, the numbers reported in the following evaluations are averages over three runs. We use COCO-style AP (i.e., AP50:5:9550:5:95{}_{\text{50:5:95}}) and AP5050{}_{\text{50}} as the evaluation metrics.

With these specifications, the total number of generated novel images is 17,714.

Leveraging synthetic data in model training. We adopt two approaches that effectively leverage synthetic data in model training: pre-training and fine-tuning (‘PT-FT’) [3, 16, 22, 37] and progressive transformation learning (PTL) [47]. ’PT-FT’ is the most widely used transfer learning method, which involves pre-training a model on a synthetic dataset and then fine-tuning it on a real dataset. ‘PTL’ is a recently proposed method that progressively expands the training set by repeatedly selecting subsets of synthetic data. Here, the selected synthetic data is first transformed to enhance realism and then added to the training set. Unless specified, PTL accuracies reported in this paper are obtained after the 5thth{}^{\text{th}} PTL iteration. In [47], only these two methods were effective in terms of accuracy among various methods of leveraging synthetic data for training.

4.1 Main Results

In Table 1, we compare the detection accuracy when using the original synthetic dataset (‘real + orig’), the pose-diversified synthetic dataset (‘real + p-div’), and neither (‘real’) for training. In all cases except for one (Vis-50 tested on the Okutama-Action dataset), leveraging synthetic data for training provides better accuracy than using only real training images. Even in this one exceptional case, using the pose-diversified synthetic dataset results in higher accuracy. This result demonstrates the impact of using synthetic data in training on improving detection performance.

Moreover, the pose-diversified synthetic dataset is generally more effective than the original synthetic dataset in increasing the detection accuracy, regardless of how the synthetic data is used for training. Interestingly, the improvement is more pronounced for the cross-domain setups evaluated on the Okutama-Action or ICG datasets. Increasing diversity of human appearances in synthetic data generally ensures higher accuracy, and the benefit becomes even greater in cross-domain tasks where the characteristics of the test set are not well captured by the training set without synthetic images. The exceptional cases (PTL & ICG test set) where accuracy is deteriorated despite using the pose-diversified dataset are discussed in a later subsection.

In addition, in most cases, PTL presents better accuracy than PT-FT, demonstrating its superiority in maximizing the impact of synthetic data. However, this superiority of PTL disappears when evaluating on the ICG test set. A subset of synthetic images used in PTL are constructed in a way that images closer (e.g., having smaller domain gap) to the training set are selected more often. The ICG test set and VisDrone training set are captured from nadir and oblique perspectives, respectively, so the human appearances in images are very different. This degraded accuracy of PTL may occur because the selected synthetic images are not representative of the ICG images. On the other hand, it is clear from the results of PT-FT that using the entire pose-diversified set in training leads to better representation ability on both same-domain and cross-domain tasks. Sample images in the pose-diversified synthetic dataset are shown in Figure 5.

4.2 Ablation Studies

We perform ablation studies on the design parameters used to construct the pose-diversified synthetic dataset. Along with the two detection accuracy metrics, we also report FID (Frénchet Inception Distance) [18], which is used to quantify the fidelity and diversity of the generated images. The VisDrone train set is used as the reference real dataset for FID calculation.

Number of final target poses for each synthetic image. Table 2(a) investigates the effects of using different number of final target poses for each original synthetic image. In this study, we scale the parameter up to 7 (i.e., 3, 5, and 7), proportionally increasing the number of total generated images. Image quality and detection accuracy do not change significantly even when the data size, controlled by changing the number of final target poses, varies. Therefore, we use 5 final target poses as default, which generates a number of images comparable to the size of the original synthetic dataset.

Post-processing. We study the effect of post-processing in Table 2(b). Filtering noisy images generally results in lower FID. The results demonstrate that noisy images are properly filtered, improving the image quality of the pose-diversified synthetic data. Moreover, it is observed that improving the image quality through this filtering process also increases the detection accuracy. On the other hand, taking occlusions into account does not improve the image quality in terms of FID, but it is useful in increasing the detection accuracy.

4.3 Scaling Behavior of Synthetic Dataset

Scaling up the dataset without ensuring sufficient diversity does not guarantee improved task performance. In this section, we explore the scaling behavior of the pose-diversified synthetic dataset to investigate the benefits of having the diversity in human appearances. In general, there are two factors in our evaluation that control the scale of the synthetic dataset used for training: i) the overall scale of the dataset and ii) the scale of the dataset selected when using PTL. We use the results shown in Figs 6 and 7 to study the scaling behavior of the pose-diversified synthetic dataset according to each of these two factors.

Scaling behavior in the data size. It could be argued that the improved performance when using the pose-diversified synthetic dataset is due to using a larger set. In Fig 6, using the pose-diversified synthetic dataset consistently presents better accuracy on VisDrone and Okutama-Action even when using the same data size as the original synthetic dataset. When tested on ICG, the accuracy with PTL is worse when using the pose-diversified synthetic dataset, similar to the main results. We will discuss this behavior in the following subsections.

Scaling behavior as PTL evolves. In Fig. 7, PTL gradually increases the number of synthetic data used for training when running more iterations. When more than 200 synthetic images are selected, a clear gap appears between using the pose-diversified set and the original set on VisDrone and Okutama-Action. It is also observed that accuracy of using the original set plateaus or decreases as more images are selected. On the other hand, when using the pose-diversified set, accuracy continues to improve for the cross-domain setups (Okutama-Action and ICG). We conclude that ensuring sufficient diversity has the effect of improving task performance, and this benefit is even greater in cross-domain settings.

Scaling behavior over PTL on ICG. In Fig. 7, it is noteworthy that when 2000 images are selected, using the pose-divsersified set yields better accuracy than using the original set on ICG. As mentioned earlier, accuracy is initially worse when using the pose-diversified dataset with fewer nadir-view images (e.g., VisDrone images) as ICG is mainly composed of nadir-view images. However, large-scale use of data that ensures sufficient diversity can eventually overcome this limitation.

5 Conclusion and Discussion

We have introduced an image generation framework that alters the original synthetic image to assume a novel pose that is not present in the original set. Using this pose-diversified synthetic dataset for training by adding the generated images to the original synthetic dataset enables machine learning models to acquire the ability to capture more human appearances, resulting in better accuracy for aerial-view human detection.

We have also studied the scaling behaviors of the synthetic dataset. In our study, the benefit of being able to represent diverse human appearances become more distinct. For example, when using more synthetic data via PTL, the detection accuracy of using our pose-diversified set continues to increase while the performance improvement plateaus when using the original set. In this section, we aim to raise some discussion points about our method.

Diversifying human poses in real data? One might consider applying our method for constructing a pose-diversified dataset to real data instead of synthetic data. However, there are some remaining challenges in applying our work to real data. For example, to create pose-diversified data using our method, some necessary information can be obtained relatively easier using the metadata provided along with the synthetic data (camera position for each input image used in target pose selection, human region masks, etc.). In addition, the image translator in our method normally requires a large-scale dataset for training, which does not align with the prerequisite settings for low-shot or cross-domain tasks that our work focuses on. Therefore, expanding real dataset in a pose-diversified manner is not within our scope but is worth considering as future work.

Broader impacts. The ‘novel poses’ in this work are in fact learned from the existing human pose dataset that is used to train the pose generator. Therefore, to construct a pose-diversified dataset using the proposed framework, the pose training set has to contain sufficiently diverse poses. In other words, if one intend to extend the proposed framework (e.g., applying to different object categories), it is crucial to make sure that such diverse pose training set exists.

In addition, aerial-view human detection enables a variety of constructive applications, such as search and rescue in humanitarian assistance and disaster relief, safe operations of UAVs, etc. However, it could also raise privacy concerns or invasion of personal spaces. As a result, it is necessary to carefully monitor the purposes for which this task is used.

Pose transformation The pose distance measurement, which uses pose transformation to align the facing direction and camera position of two input poses, is employed in several aspects within our method. While the main manuscript outlines the general strategy of pose transformation, we provide specific details here regarding the pose transformation implemented in each scenario.

The pose distance measurement is used in the following scenarios:

  • •

    Building the novel pose set (3D pose vs. 3D pose). To ensure pose diversity within the novel pose set, any generated poses are excluded if they closely resemble existing poses in the set in terms of pose distance. Here, the poses within the novel pose set are represented by the 3D locations of human keypoints. When comparing a generated pose to each pose in the set, the generated pose undergoes a transformation to align its facing direction with that of the pose being compared, where the facing direction is defined by a neck-to-nose vector. Since the pose generator is trained using poses captured from a ground view, two generated poses being compared are assumed to have the same camera position on the ground. Therefore, no further transformation is required.

  • •

    Selecting target poses (2D pose vs. 3D pose). In the process of target pose selection, we calculate the pose distance between each input pose in the original synthetic data, represented in 2D keypoint positions, and every pose in the novel pose set, represented in 3D keypoint positions. The camera position corresponding to the input pose can be extracted from the metadata of the original synthetic data. Therefore, we first transform poses in the novel pose set to align with the camera positions of the input pose. We then adjust these poses to match the facing direction of the input pose using the neck-to-nose vector. After that, the transformed poses are projected onto a 2D plane for the computation of pose distance.

  • •

    Filtering noisy images (2D pose vs. 2D pose). In the process of noisy image filtering, we compute the pose distance for each generated image, measuring the disparity between the target pose used during generation and the estimated pose. As these two poses correspond to the same image, no transformation is needed.

Image generation. The overview of image generation is shown in Fig. 8. To minimize potential noise coming from the background during the generation of novel images, we adopt a strategy of processing the human and background regions independently. Segmentation of the human region can be achieved through the utilization of the segmentation mask provided within the Archangel-synthetic dataset [48], serving as the original synthetic dataset in our scenario.

The human region where the background is excluded is filled with a mono color (in our case, gray) to facilitate the creation of the segmentation mask after image generation. With a given target pose, this human image undergoes alterations to ensure that the human in the image assumes the target pose, using the source-to-target image translator (‘image tr.’). The segmentation mask for the human within the generated image is then constructed by capturing pixels that deviate from the pre-defined mono color. To mitigate artifacts in the segmentation mask, we apply a series of morphological operations, including four iterations of dilation followed by two iterations of erosion, using a kernel size of 3×3333\times 3.

The human region, segmented from the generated image using the segmentation mask, is then resized using a randomly selected resizing factor, while maintaining a fixed aspect ratio. To elaborate, we randomly select a human image from the Archangel-synthetic dataset and determine the resizing factor such that the longer side of the human region in the generated image matches that of the human region in the randomly selected image. Subsequently, the generated and resized human region is reinserted into the background at a random location, and the original human region is inpainted. To introduce occlusion to the generated image, we add two generated human regions onto the image during post-processing.

Training details. The proposed framework comprises three types of model training: pose generator training, image translator training, and detector training (i.e., PT-FT and PTL). In this section, we delve into additional training details that were not addressed in the main manuscript. All the models were trained on a server with 4 NVIDIA GeForce RTX 2080 Ti GPUs.

  • •

    Pose generator training. We built our pose generator based on the official implementation of MDM [52]. Without further specifications, we maintained all the settings and training strategies as defined in the original implementation. The pose generator was trained on the Human3.6M dataset [19], which was preprocessed according to the procedure outlined in VideoPose3D [41]. We trained the pose generator for 600k iterations with a batch size of 256, a learning rate of 0.0001, and 1k diffusion steps. Other details, including network architecture and loss function, are provided in the main manuscript (Sec. 3.1).

  • •

    Image translator training. We utilized the official NTED model [45] as our image translator, which had been pre-trained on the DeepFashion dataset [34], with an input human image size of 256×176256176256\times 176. We fine-tuned the pre-trained image translator on the Archangel-synthetic dataset for 200 epochs with a batch size of 4. During fine-tuning, we adjusted the learning rate to be 10 times smaller (i.e., 0.0002) than the default setting used for pre-training. Additionally, the generative adversarial loss was incorporated after 20k training iterations. Default settings were applied if no further specifications were provided.

  • •

    Detector training. In this work, we utilized two different methods that leverage synthetic data in detector training: PTL and PT-FT. For PTL, we adhered to the official implementation [47], which trained a variant of RetinaNet [31] in a progressive manner, using default hyper-parameters. It’s important to note that the original PTL framework only considers the case when there is only a single virtual character present in each synthetic image. However, in our pose-diversified synthetic dataset, there might be two virtual characters in a single image. To address this issue, when calculating the domain gap (i.e., Mahalanobis distance) for each synthetic image containing two characters, we computed the domain gaps separately for each character and represented the synthetic image based on the character with the smaller domain gap. Moreover, if a synthetic image with two characters was selected during a PTL iteration, both characters were involved in the syn2real transformation. For PT-FT, we adopted the same detector architecture and training setups as specified in PTL, with the exception of omitting the progressive aspect. The detector was pre-trained on the synthetic dataset for 6k iterations, followed by fine-tuning on the real dataset for 600 iterations.

Appendix 0.B Additional Experiments

To ensure the general applicability of our primary analysis in the main manuscript (Sec. 4.1), we perform additional experiments within the same setup, using different training datasets. Here, we replace the training domain from VisDrone with Okutama-Action or ICG (Tab. 3). Consistent with our primary analysis, using the pose-diversified synthetic dataset generally results in higher accuracy than using the original synthetic dataset, with one exception: when leveraging synthetic data for training using PTL and testing trained detectors on the ICG test set.

Meanwhile, two cases exhibit different trends from the main results obtained with the VisDrone dataset for training. Both cases arise during testing on the ICG dataset, which primarily consists of nadir-view images. First, when using PT-FT in the Oku-50 setting, utilizing pose-diversified synthetic data presents lower accuracy. In this case, the adverse effects of using pose-diversified synthetic data with relatively fewer nadir-view images become more severe even though the data is used for pre-training but not for fine-tuning. Second, in the ICG-20 and ICG-50 settings with PTL+ICG, the AP5050{}_{\text{50}} accuracy improves when using the pose-diversified synthetic dataset, while the AP accuracy remains lower. The lack of nadir-view images during training results in degraded accuracy in precise detection within the ICG test set. However, it appears that this deficiency can be compensated to some extent by ensuring sufficient pose diversity in the data when using a lenient detection criterion (i.e., IoU threshold = 50%).
