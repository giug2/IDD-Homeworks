Abstract.

Federated recommendation systems are essential for providing personalized recommendations while protecting user privacy. However, current methods mainly rely on ID-based item embeddings, neglecting the rich multimodal information of items. To address this, we propose a Federated Multimodal Recommendation System, called FedMR. FedMR uses a foundation model on the server to encode multimodal item data, such as images and text. To handle data heterogeneity caused by user preference differences, FedMR introduces a Mixing Feature Fusion Module on each client, which adjusts fusion strategy weights based on user interaction history to generate personalized item representations that capture usersâ€™ fine-grained preferences. FedMR is compatible with existing ID-based federated recommendation systems, improving performance without modifying the original framework. 1. Introduction

Under privacy protection regulations, such as GDPR (Voigt and VonÂ dem Bussche, 2017), that emphasize user data privacy, Federated Recommendation Systems (FedRec) have become an essential technology in modern life (Sun etÂ al., 2024). However, in these systems, since each client typically only has access to a single userâ€™s interaction history, data sparsity is a common issue that degrades recommendation performance (Li and Long, 2024). Modeling item features as shared information across clients can effectively mitigate the impact of data sparsity. Many studies (Zhang etÂ al., 2023c; Li etÂ al., 2023a, 2024b) have shown that personalized item representations can significantly improve recommendation results. Despite items often containing rich multimodal information (Zhou etÂ al., 2023b), such as such as text and images, most existing FedRec methods (Zhang etÂ al., 2023d, 2024d, 2024f) still rely mainly on ID embeddings, underutilizing the potential of multimodal data. Thus, generating personalized item representations from multimodal data while ensuring user privacy remains a significant challenge.

As a potential solution, Multimodal Recommendation Systems (MMRec) integrate data from different modalities, significantly enhancing recommendation performance (He and McAuley, 2016; Wei etÂ al., 2019; Zhou etÂ al., 2023a). Unlike single-modal systems, MMRec can capture hidden relationships and complementary information between various modalities, leading to a better understanding of user preferences and item characteristics (Zhou etÂ al., 2023b). For example, MMRec methods (Wei etÂ al., 2019; Wang etÂ al., 2021; Tao etÂ al., 2022) combine video frames, audio tracks, and item descriptions when users interact with online platforms, providing a more accurate representation of user preferences. To effectively handle multimodal data, MMRec often uses Foundation Models (FM) pretrained on large-scale datasets, which possess powerful feature extraction and representation capabilities (Bommasani etÂ al., 2021). By applying FMs, MMRec can efficiently extract and integrate information from multiple modalities, capturing commonalities and complementary relationships between them (Radford etÂ al., 2021a; Cui etÂ al., 2022; Geng etÂ al., 2022, 2023; Wu etÂ al., 2024). However, these centralized MMRec methods rely on servers to aggregate user data, digital activities, and preferences to train accurate models (Liu etÂ al., 2024), which raises privacy concerns (Chai etÂ al., 2020; Li etÂ al., 2023a).

To address these challenges, Federated Multimodal Recommendation Systems (FedMMRec) have emerged, which integrates MMRec and introduces the capability of processing multimodal information into FedRec. By utilizing multimodal data and user-specific information, such as profiles and interaction history, locally on each client to generate personalized recommendations and aggregating model parameters at the server, FedMMRec enhances recommendation performance while preserving user privacy (Feng etÂ al., 2024). To the best of our knowledge, FedMMRec is still in the early stages of research, and related studies are limited. FedMMR (Wu etÂ al., 2024) adopts a model-splitting strategy where the computationally intensive components are handled by the server, while clients perform lightweight collaborative filtering. AMMFRS (Feng etÂ al., 2024) introduces an attention mechanism on the client side to perform weighted fusion of multimodal data, effectively distinguishing and utilizing the contributions of different modalities. These methods not only protect user privacy but also effectively integrate multimodal features to improve recommendation accuracy and personalization. However, they rely on fixed multimodal fusion strategies, neglecting the personalized representation of item features on clients, which limits their ability to adapt to diverse user needs. Although AMMFRS uses an attention mechanism to distinguish the importance of different modalities, it lacks adaptability to the diversity of user data, resulting in limited flexibility and accuracy of fusion strategies (Niu etÂ al., 2021).

Main Contribution. To integrate multimodal item information into FedRec while ensuring user privacy, we propose a novel Federated Multimodal Recommendation system (FedMR). FedMR uses a foundation model on the server to encode multimodal item data (e.g., images, text) into feature embeddings, and constructs a global item ID embedding to capture structural features among items. On each client, FedMR introduces a Mixing Feature Fusion Module that dynamically adjusts the importances of fusion strategies based on user interaction history, combining multimodal and ID embeddings to model fine-grained personalized item representations for users. Thus, FedMR seamlessly integrates multimodal information with existing ID-based FedRec methods, improving the flexibility and accuracy of recommendations without altering the existing framework, and effectively handles data heterogeneity across clients, providing more personalized recommendations.

2. Related Work

2.1. Federated Recommendation Systems

Federated recommendation systems (FedRec) aim to provide personalized recommendations while ensuring user privacy and data security (Yang etÂ al., 2020). In recent years, FedRec has made significant progress in areas such as rating prediction (Lin etÂ al., 2020a, b; Liang etÂ al., 2021; Lin etÂ al., 2022; Zhang etÂ al., 2023d; Li etÂ al., 2024a), top-k recommendation (Ammad-Ud-Din etÂ al., 2019; Perifanis and Efraimidis, 2022; Luo etÂ al., 2022; Zhang etÂ al., 2023b; Dong etÂ al., 2022; Li etÂ al., 2022a; Qu etÂ al., 2023), and addressing the cold-start problem (Liu etÂ al., 2021b; Meihan etÂ al., 2022; Wahab etÂ al., 2022; Zhang etÂ al., 2024c, d). However, most existing FedRec methods (Huang etÂ al., 2020; Flanagan etÂ al., 2021; Wu etÂ al., 2022; Zhu etÂ al., 2022; Imran etÂ al., 2023; Yuan etÂ al., 2023a, 2024) rely solely on ID embeddings to model item representations, overlooking the rich multimodal information (e.g., titles and covers) that items may have. AMMFRS (Feng etÂ al., 2024) and FedMMR (Wu etÂ al., 2024) apply multimodal learning (BaltruÅ¡aitis etÂ al., 2018) to FedRec. AMMFRS uses an attention mechanism (Niu etÂ al., 2021) to weight and fuse multimodal data, emphasizing the contribution of different features. In contrast, FedMMR assigns the computationally heavy multimodal learning tasks to the server and introduces a secure cross-client convolution strategy to capture high-order interaction information using secret sharing techniques while preserving privacy. Unlike methods that focus only on personalizing user information, PFedRec (Zhang etÂ al., 2023c) proposes a dual personalization mechanism that optimizes personalization for both users and items, integrating it into existing FedRec frameworks. Building on this, FedRAP (Li etÂ al., 2023a) introduces additive personalization, which refines the modeling of user and item features. FedDAE (Li etÂ al., 2024b) employs a dual-encoder variational autoencoder to adaptively fuse shared and personalized item knowledge through a personalized gating network. While these methods achieve personalized item representations, they still mainly rely on ID embeddings, failing to fully utilize the rich multimodal information of items.

2.2. Multimodal Recommendation Systems

Unlike traditional Recommendation Systems (RecSys) that rely primarily on user-item interaction data (such as clicks or ratings) to capture user behavior (Yu etÂ al., 2023), Multimodal Recommendation systems (MMRec) aim to improve the accuracy of recommendations and enhance user experience by utilizing multiple modalities of data , such as text, images, audio, and video (Zhou etÂ al., 2023b). As unique identifiers for items, item IDs not only contain semantic information about the items but also capture the characteristics of neighboring items (Yuan etÂ al., 2023b; Liu etÂ al., 2023). Therefore, in MMRec, item IDs are often used as auxiliary information and combined with multimodal data to enhance item representations(Wang etÂ al., 2024; Zhang etÂ al., 2024e). This integration helps capture a more comprehensive understanding of usersâ€™ interests and preferences, ultimately improving the quality of recommendations. A typical MMRec training process generally involves feature extraction and modality fusion (He and McAuley, 2016; Wei etÂ al., 2019; Wang etÂ al., 2021; Tao etÂ al., 2022). Feature extraction (Guo etÂ al., 2019) aims to obtain representative features from multiple modalities to better capture the attributes and preferences of users and items. Modality fusion (Gao etÂ al., 2020) focuses on effectively integrating data from different modalities to enhance the overall performance of the MMRec and the relevance of its results. The recommendation models (Tao etÂ al., 2020; Wu etÂ al., 2021; Zhou and Shen, 2023) combine multimodal data with user-item interaction data to predict user preferences and provide personalized recommendations.

2.3. Foundation Models

Foundation Models (FM) are often pre-trained on large-scale unlabeled datasets for general purposes, that can be adapted to a variety of downstream tasks through appropriate fine-tuning, such as task-specific training (Bommasani etÂ al., 2021). This makes them highly generalizable and adaptable across different applications. They are then adapted to specific recommendation tasks through methods such as representation transfer (Du etÂ al., 2020; Liu etÂ al., 2021a, 2022; Wei etÂ al., 2023), model fine-tuning (Zhang etÂ al., 2021; Li etÂ al., 2022b, 2023b; Sun etÂ al., 2023; Wang etÂ al., 2023), adapter tuning (Hou etÂ al., 2022; Bian etÂ al., 2023; Geng etÂ al., 2023; Fu etÂ al., 2024), and prompt tuning (Shin etÂ al., 2020; Li and Liang, 2021; Zhou etÂ al., 2022; Zhang and Wang, 2023; Xu etÂ al., 2024). Some studies have explored the use of FM to enhance the content-awareness of MMRec, improving content understanding and generation by leveraging alignment and generation techniques across multiple modalities. CLIP (Radford etÂ al., 2021a; Wu etÂ al., 2024) uses contrastive learning (Le-Khac etÂ al., 2020) between language and images to obtain multimodal representations, capturing the matching relationships between users and items. M6-Rec (Cui etÂ al., 2022) builds a recommendation system based on large-scale multimodal pre-trained models specifically designed for the e-commerce domain. P5 (Geng etÂ al., 2022) transforms the recommendation task into a natural language processing task, generating personalized recommendations through prompts. VIP5 (Geng etÂ al., 2023) leverages pre-trained multimodal models to capture complementary information from visual and textual data, providing more personalized and relevant recommendations to users. FedPA (Zhang etÂ al., 2024c) and TransFR (Zhang etÂ al., 2024b) integrate FM with FedRec. FedPA introduces a personalized adapter to fine-tune the FM, allowing each client to learn individual user preferences in a lightweight manner. In contrast, TransFR leverages FMs to generate universal textual representations, replacing traditional ID-based item representations and enhancing the systemâ€™s adaptability to new domains and its ability to handle cold-start issues.

Removing any one of these disrupts the balance, making it difficult for the model to fully represent user preferences and item characteristics. Additionally, different methods depend on different modalities to varying extents. For instance, FCF and PFedRec are more dependent on visual information, whereas FedAvg is less reliant on ID features. These differences in dependency lead to varying levels of performance decline when specific modalities are removed. Further experimental results are provided in the appendix.

5.6.3. Ablation Study on Fusion Strategy

Fig. 3 presents the HR@50 and NDCG@50 evaluation results for five used FedRec methods using different feature fusion strategies with FedMR on the KU dataset. The results indicate that different fusion strategies significantly impact performance. Notably, the Mix strategy, which dynamically combines multiple fusion methods based on user interaction history, outperforms others, while the Sum strategy shows poor performance. This trend is consistent across all metrics.

The Sum strategy simply adds different modality features without considering their complex interactions and inter-relationships. Since each modalityâ€™s importance varies, simple addition can lead to information loss or imbalanced weighting, making it difficult to capture relationships effectively. Additionally, RecSys often face a long-tail distribution, where a few popular items dominate user interactions (Park and Tuzhilin, 2008; Yin etÂ al., 2012). The simplicity of the Sum strategy may lead to overfitting on popular items, resulting in poor performance on less interacted items and affecting overall model performance.

In contrast, the MLP strategy uses fully connected networks to learn nonlinear modality combinations, which better captures complex interactions between item modalities and ID features. However, its effectiveness is limited by network depth and the number of parameters. On the other hand, the Gate strategy learns adaptive weights to dynamically filter the contribution of each modality, optimizing the importance of different features.

The Mix strategy integrates all fusion methods and adjusts their weights dynamically based on user interaction history, allowing the model to capture more fine-grained personalized item representations. This approach enables more accurate matching of user preferences and effectively captures complex relationships between multimodal features, leading to superior performance.

This study highlights the importance of personalized item embeddings in FedRec. In FedMR, each client dynamically adjusts fusion strategy weights through the Mixing Feature Fusion Module, based on user interaction history, to generate personalized item embeddings, enhancing personalization and accuracy while maintaining user privacy, thus improving overall system performance.

5.6.4. Privacy Protection

Table 4 presents the experimental results of the used FedRec methods combined with FedMR, comparing the performance with and without adding Gaussian noise (mean=0, variance=0.1) to the gradients of the global components uploaded to the server, to protect user privacy. The experiments are conducted on the KU dataset during feature fusion. The results indicate that all methods experienced some performance degradation after adding noise, particularly on the NDCG metric, which is more sensitive to ranking accuracy. This is because noise disturbs the gradient precision, which affects the modelâ€™s ability to update effectively.

Among the methods, FedNCF is the most sensitive to noise, with HR@50 dropping by 14.33% and NDCG@50 by 31.03%. As a complex neural network-based model, FedNCF depends on numerous parameters to capture nonlinear features. The introduction of noise disrupts these feature interactions, resulting in a significant decline in performance. In contrast, FedAvg and FedRAP show greater robustness to noise, with only minimal performance degradation. This suggests that these methods are more effective at maintaining recommendation accuracy while ensuring privacy protection.

In conclusion, although adding noise helps protect user privacy, its impact varies based on model complexity and feature interaction mechanisms. Simpler models like FedAvg and adaptive models like FedRAP are more resilient to noise, while complex neural network models like FedNCF experience more significant performance drops due to the noise interference.

6. Conclusion

This paper presents FedMR, a federated multimodal recommendation system that incorporates a mixing feature fusion module on each client. This module dynamically adjusts fusion weights based on user interaction history, generating personalized item representations that capture fine-grained user preferences, thereby enhancing personalized recommendations. It allows flexible modification of fusion strategies and supports different modalities of item data as needed. Experimental results on four multimodal recommendation datasets show that FedMR performs well and is compatible with existing ID-based methods without requiring changes to the underlying framework. This study offers new directions for the development of federated recommendation systems, particularly in the application of multimodal data and foundation models.

Appendix D Limitations and Future Work

While FedMR has shown promising improvements in federated multimodal recommendation systems, several limitations remain:

  1. (1)

    Data Heterogeneity: Considering the federated settings, the distribution of user data varies significantly. Although FedMR mitigates this issue by integrating multimodal information, its performance is still limited when handling highly heterogeneous data. Future work could focus on introducing more robust personalization models or adaptive aggregation strategies to better address the differences between users.

  2. (2)

    Communication Overhead: Although FedMR reduces some communication costs by encoding multimodal data, processing multimodal features still results in high computational and communication costs. Future research could explore optimizing communication protocols or using model compression techniques to further minimize these costs.

  3. (3)

    Limited Flexibility in Multimodal Fusion: While FedMR supports dynamic adjustments to its fusion strategies, its current approach may lack flexibility when dealing with complex user behaviors. Future work could explore more adaptive and layered fusion methods to better capture the diverse preferences of users.

In conclusion, future developments could enhance the overall performance of federated recommendation systems by improving the handling of data heterogeneity, reducing communication costs, and optimizing multimodal fusion strategies.
