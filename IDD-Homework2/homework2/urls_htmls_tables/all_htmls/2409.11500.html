1 Synthetic Data Quality Human Evaluation Guidelines

The primary purpose of the annotation is to assess the quality of synthetically generated user-agent conversations. Each dialog is grounded on one or more (up to five) document passages. Evaluation should be carried out on the basis of the information available from the documents. Evaluation criteria are summarized in Table 1. Note the relationship between query answerability and response correctness as follows:

  • •

    If the query is unanswerable, then the correct response is I do not know/No information is available in the given document

  • •

    If the query is answerable, then the correct response is factual information from the given document.

In determining the correctness of a response, follow the procedure specified below:

  1. 1.

    Check to see if the query consists of a single clause or multiple clauses

  2. 2.

    If the query consists of a single clause, then look for possible answers in the documents

    • •

      If the possible answers overlap with the provided response, then the response is correct.

    • •

      If the response is correct w.r.t. the query and contains some extra information, the response is deemed correct as long as the extra information does not contradict the main answer.

  3. 3.

    If the query consists of multiple clauses, repeat 2 until all clauses in the query have been answered
