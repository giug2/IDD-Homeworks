Leveraging feature communication in federated learning
for remote sensing image classification

Abstract

In the realm of Federated Learning (FL) applied to remote sensing image classification, this study introduces and assesses several innovative communication strategies. Our exploration includes feature-centric communication, pseudo-weight amalgamation, and a combined method utilizing both weights and features. Experiments conducted on two public scene classification datasets unveil the effectiveness of these strategies, showcasing accelerated convergence, heightened privacy, and reduced network information exchange. This research provides valuable insights into the implications of feature-centric communication in FL, offering potential applications tailored for remote sensing scenarios.

1 Introduction

Remote sensing applications often grapple with the challenge of processing vast amounts of data distributed across various geographical locations. As the demand for sophisticated image classification models grows, the need for efficient and privacy-preserving methodologies becomes imperative. If we collect data in remote sensing and train a centralized model, we will encounter privacy issues. On the contrary, if we train a local model, we will face challenges related to non-uniform distribution and insufficient data across devices.

Federated Learning (FL) emerges as a promising solution, allowing decentralized devices to collaboratively train machine learning models while keeping raw data localized. FL has various approaches such as FedAVG, FedSGD, FedBN, etc. [1] and can even adopt a peer-to-peer (P2P) [2] model instead of the traditional client-server architecture. FedAVG [3], or Federated Averaging, has been a key algorithm in the domain of FL. It enables multiple decentralized devices or clients to collaboratively train a global model without sharing raw data. This approach ensures privacy by keeping raw data localized while allowing collaborative model training across a distributed network. FedAVG is renowned for its efficiency, simplicity and longevity, acting as a baseline for other algorithms to develop.

Remote sensing problems usually rely on devices that may be located far from the ground, limiting the information being communicated [4]. Thus, there is a crucial need for FL methods that function on small amount of information. In this context, FedAVG has been primarily adopted [5] for aggregating models across devices.

In this paper, we explore various FL communication strategies for remote sensing image classification based on the well-known FedAVG algorithm and study the trade-off between reducing data communicated and performance. In particular, we propose sending and receiving only the average feature vectors for each categories of interest which incurs minimal information. As such, one device could gain insights from the features extracted by the other clients without explicitly synchronizing the parameters between separated training device. The idea that a model can learn from feature representations instead of using network parameters has been widely applied between images and text [6] as CLIP [6] employs Vision Transformer (ViT) to learn feature representations of text extracted from BERT, aiming to maximize the similarity of (image, text) pairs using a contrastive loss. To that end, the Large Margin Cosine Loss [7] is employed to solve the data imbalance problem and allow faster training convergence. Regarding to improve network performance, we combine the FedAVG algorithm with average feature communication and formulate classification as a retrieval problem, obtaining the best of both worlds, high performance with fast convergence.

2.1 Revisiting FedAVG

This section briefly revisits and explains the FedAVG algorithm¬†[3]. For more details, we refer interested readers to the original work. FedAVG is a server-client communication model. A client Cisubscriptùê∂ùëñC_{i} is an individual device training a local model using a separate set of data, disjoint with the others and the server SùëÜS is the central entity responsible for maintaining a global model by aggregating the local models contributed from the clients. The training process composes of tùë°t rounds of training the clients locally and communicating (for synchronization) between each of them and the server.

In FedAVG, each client, after kùëòk epochs of local training, transmits all the trained parameters, also called weights or, collectively, model, to the server. The server‚Äôs and all clients‚Äô models are, then, updated with the average model. The process is described in Algorithm¬†1.

For evaluation, the server‚Äôs global model is used and the prediction score from the softmax layer is used to determine the predicted category of an input image.

Feature communication

LMCL is similar to the cross-entropy loss with softmax function but without requiring the head layer weights WjTsuperscriptsubscriptùëäùëóùëáW_{j}^{T}. Instead, it replaces the weight by another feature vector and measures the similarity between them. Therefore, is it possible to have clients exchange only the representations of features for each label with the server, without transmitting their weights. Algorithm¬†2 describes the process.

Different from the vanilla FedAVG, each client does not send the entire trained parameters to the server but only the averages of the feature vectors extracted by the backbone with shape [ncat,d]subscriptùëõcatùëë\left[n_{\text{cat}},d\right] (where ncatsubscriptùëõcatn_{\text{cat}} is the number of categories, and dùëëd is the embedding dimension of the features). The server receives ncatsubscriptùëõcatn_{\text{cat}} vectors from all the clients and compute ncatsubscriptùëõcatn_{\text{cat}} average features across the clients. Figure¬†1 shows a visualization.

In this way, only the average features for each label, which serve as representations of each class across all the clients, are transferred, maintaining minimal information communicated. The LMCL compels the clients‚Äô models to learn a representation that is close to these common representations.

In this algorithm, the Server does not maintain an aggregated global model but only act as an intermediary to synchronize all the clients. For evaluation, we run all client models through the same validation data and average their performance.

2.4 Retrieval-based classification

So far, classification prediction is provided by the softmax score of either the server‚Äôs or clients‚Äô models and feature communication is only performed during the training process. In this section, we propose transferring the client‚Äôs extracted features for making prediction during the deploying stage. To that end, the classification problem is reformulated as a retrieval problem and presented in Algorithm¬†5.

The idea requires a preparation step before prediction can be made, which includes re-passing all training images through each client‚Äôs backbone to extract the feature vectors, similar to step¬†3 in Algorithm¬†2, which are distributed to all other clients via the server. Each client classifies an input image by extracting its feature using the backbone model and retrieving the label of the majority of the nearest training features in the embedding space.

Note the difference with Algorithm¬†2‚Äôs idea where the extracted feature of each training image is transferred instead of the per-class average vectors. As this could be expensive for large training sets, we also study the case where the average vectors (as in lines¬†4,¬†5 of Algorithm¬†2) are communicated and indicate it as Algorithm¬†6. This will reduce the amount of information communicated across the network while being more secure [10] due to the low risk of data being exposed in case the features can be reversed.

4 Conclusion

This work has proposed and compared different FL approaches, including communicating only with features, combining features with pseudo-weights, and utilizing both weights and features for training and testing. Our experiments on remote sensing two datasets showed the effectiveness of these approaches by highlighting faster convergence and reducing the amount of information transmitted over the network. This research contributes to exploring the impacts of features in various communication strategies in FL, particularly in the context of remote sensing applications.
